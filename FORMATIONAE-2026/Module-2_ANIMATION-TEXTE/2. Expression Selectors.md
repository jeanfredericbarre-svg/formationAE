## L'Expression Selector

L'Expression Selector est le troisième type de sélecteur disponible dans les Text Animators, après le Range Selector et le Wiggly Selector. Il remplace la logique interne du sélecteur par **une expression que tu écris toi-même**.

**Pour l'ajouter :** dans un Animator existant, clique sur **Ajouter** → **Sélecteur** → **Expression**.

L'expression que tu écris dans ce sélecteur doit retourner une valeur **entre 0 et 1** — ou entre 0% et 100% selon les unités choisies. Cette valeur représente l'**intensité** avec laquelle l'Animator s'applique à chaque caractère.

```
Retourner 1   (ou 100%)  →  le caractère est pleinement affecté par l'Animator
Retourner 0   (ou 0%)    →  le caractère n'est pas du tout affecté
Retourner 0.5 (ou 50%)   →  le caractère est à moitié affecté
```

> 💡 C'est là que la puissance des expressions rencontre la puissance des Text Animators. Au lieu d'une plage figée ou d'un wiggle aléatoire, tu peux écrire **n'importe quelle logique** pour décider quelle lettre est affectée, et à quelle intensité.

## textIndex — l'index du caractère

Dans une expression de sélecteur, la variable `textIndex` est **disponible automatiquement**. Elle retourne le numéro du caractère en cours d'évaluation — AE évalue l'expression une fois par caractère, et `textIndex` change à chaque évaluation.

```javascript
textIndex   // 1 pour le premier caractère, 2 pour le deuxième, etc.
            // Attention : commence à 1, pas à 0 — contrairement aux Arrays JS
```

**Exemple simple :** affecter uniquement les caractères pairs.

```javascript
// Dans l'Expression Selector :
if (textIndex % 2 === 0) {
  1   // caractère pair → pleinement affecté
} else {
  0   // caractère impair → non affecté
}
```

## textTotal — le nombre total de caractères

`textTotal` retourne le **nombre total de caractères** dans le calque texte. Combiné à `textIndex`, il permet de calculer la **position relative** de chaque caractère — exactement comme la normalisation de `time` vue dans le Module 3.

```javascript
textIndex / textTotal
// → 1/10 = 0.1 pour le premier caractère d'un mot de 10 lettres
// → 10/10 = 1.0 pour le dernier caractère
```

Cette valeur normalisée entre 0 et 1 peut directement alimenter `linear()` ou `ease()`.

## Exemples pratiques

### Entrée en cascade — délai croissant par lettre

Chaque lettre commence son animation légèrement après la précédente. L'effet : le mot se déploie de gauche à droite.

```javascript
// Dans l'Expression Selector (combiné avec un Animator Opacité à 0%) :
var delai = 0.08;                          // délai en secondes entre chaque lettre
var debut = (textIndex - 1) * delai;       // quand cette lettre commence
var fin = debut + 0.3;                     // durée de la transition (0.3s)

linear(time, debut, fin, 1, 0)
// retourne 1 (lettre invisible) jusqu'à "debut"
// puis passe progressivement à 0 (lettre visible) jusqu'à "fin"
```

> 💡 Note l'inversion `1, 0` et non `0, 1` : l'Animator applique l'opacité à 0%, donc une intensité de 1 = opacité à 0% (invisible), et une intensité de 0 = opacité normale (visible).

### Progression par position — du centre vers les bords

Les lettres du centre apparaissent en premier, puis l'effet se propage vers les extrémités.

```javascript
var centre = (textTotal + 1) / 2;           // position du caractère central
var distance = Math.abs(textIndex - centre); // distance au centre
var distMax = centre - 1;

linear(distance, 0, distMax, 0, 1)
// caractère central → distance = 0 → intensité = 0 (pleinement affecté en premier)
// caractères extrêmes → distance max → intensité = 1 (non affectés)
```

### Alternance régulière

Une lettre sur trois est affectée — effet de peigne.

```javascript
(textIndex % 3 === 0) ? 1 : 0
// l'opérateur ternaire : si textIndex est divisible par 3 → 1, sinon → 0
```
