<span id="lien-calque"></span>
## sourceText lié à un autre calque

La technique la plus directe et la plus légère : lier le `sourceText` d'un calque texte au `sourceText` d'un autre calque via une expression pure, sans script externe.

**Cas d'usage typique :** un calque "source" masqué hors champ qui contient le texte brut, et plusieurs calques visuels (version grande, version petite, version colorée) qui le lisent tous depuis cette source unique. Modifier le texte à un seul endroit met à jour toute la comp.

```javascript
// Sur le calque texte "cible", propriété Texte source :
const source = thisComp.layer("SOURCE_TEXTE");
source.text.sourceText
```

Pour accéder au texte d'un calque dans une autre composition :

```javascript
// Lire le sourceText depuis une précomp
const precomp = comp("DONNEES");
precomp.layer("NOM_INTERVENANT").text.sourceText
```

> 💡 Cette technique fonctionne aussi pour synchroniser un sous-titre avec un calque de titre : le sous-titre lit le `sourceText` du titre principal et le transforme (mise en majuscules, troncature, etc.) via une expression.

**Transformer le texte lu :**

```javascript
// Récupérer le texte source et le passer en majuscules
const source = thisComp.layer("SOURCE_TEXTE").text.sourceText;
source.toString().toUpperCase()
```

```javascript
// Afficher uniquement les 20 premiers caractères
const source = thisComp.layer("SOURCE_TEXTE").text.sourceText;
source.toString().substring(0, 20)
```

## Le principe : séparer le contenu de l'animation

Dans un projet data-driven, le contenu (textes, noms, chiffres) et l'animation (timing, effets, mise en forme) sont deux responsabilités distinctes. Les confondre dans un même fichier AE crée des projets difficiles à maintenir : chaque mise à jour de texte nécessite de rouvrir AE, de naviguer dans la timeline, de modifier manuellement chaque calque.

La séparation repose sur un principe simple :

```
Fichier de données (JSON, CSV)
        ↓
  Script ExtendScript (.jsx)
        ↓
  Calques texte dans AE  ←  Animation inchangée
```

Le script fait le lien entre les données et les calques. L'animateur ne touche plus au contenu, seulement à la mise en forme et au timing. Un chef de projet peut mettre à jour les textes sans connaître AE.

<span id="extendscript"></span>
## ExtendScript : lire un fichier JSON

ExtendScript est le moteur de scripting d'Adobe (basé sur JavaScript ES3). Il permet d'accéder au système de fichiers, de lire des données externes, et de manipuler n'importe quelle propriété d'un projet AE.

**Structure du fichier JSON de données :**

```json
[
  { "nom": "Marie Dupont", "titre": "Directrice artistique" },
  { "nom": "Paul Martin", "titre": "Chef opérateur" },
  { "nom": "Sofia Reyes", "titre": "Monteuse" }
]
```

**Script ExtendScript pour peupler les calques :**

```javascript
// Script à lancer depuis Fichier → Scripts → Exécuter le fichier de script
// Le fichier JSON doit être au même niveau que le projet AE

const projet = app.project;
const comp = projet.activeItem;  // la comp active dans AE

// Ouvrir le fichier JSON
const fichierJson = File(projet.file.parent.fsName + "/generique.json");
fichierJson.open("r");

// Lire et parser le contenu
const contenu = fichierJson.read();
fichierJson.close();
const donnees = JSON.parse(contenu);

// Boucler sur les entrées et assigner les textes
for (let i = 0; i < donnees.length; i++) {
    const entree = donnees[i];
    const indexCalque = i + 1;  // les calques sont indexés à partir de 1 dans AE

    // Cibler les calques par nom conventionnel
    const calqueNom   = comp.layer("NOM_" + indexCalque);
    const calqueTitre = comp.layer("TITRE_" + indexCalque);

    if (calqueNom)   calqueNom.property("Source Text").setValue(entree.nom);
    if (calqueTitre) calqueTitre.property("Source Text").setValue(entree.titre);
}

alert("Générique mis à jour — " + donnees.length + " entrées chargées.");
```

> 💡 `projet.file.parent.fsName` retourne le chemin du dossier contenant le projet AE. Placer le JSON dans ce dossier garantit que le script le trouve quelle que soit la machine.

**Pour lancer le script :** `Fichier → Scripts → Exécuter le fichier de script` → sélectionner le `.jsx`.

## Le workflow Bodymovin et Lottie

**Bodymovin** est un plugin AE (open source) qui exporte une composition sous forme de fichier JSON Lottie — un format d'animation vectorielle lisible par les navigateurs et les applications mobiles.

L'intérêt pour le texte dynamique : une animation Lottie peut exposer ses calques texte comme des **champs modifiables** côté web ou mobile, sans retoucher AE ni réexporter.

**Mise en place dans AE :**

1. Nommer les calques texte de façon explicite (`TITRE`, `SOUS_TITRE`, `DATE`)
2. Dans Bodymovin, activer l'option **Glyphs** ou **Characters as shapes** selon le besoin de personnalisation des polices
3. Exporter en JSON

**Modifier le texte côté web (Lottie Web) :**

```javascript
// Exemple avec la bibliothèque lottie-web
const animation = lottie.loadAnimation({
    container: document.getElementById('animation'),
    renderer: 'svg',
    animationData: animationJson
});

// Modifier un calque texte après chargement
animation.addEventListener('DOMLoaded', () => {
    const elements = animation.renderer.elements;
    // Cibler le calque par son nom et modifier le texte
});
```

> 💡 Le workflow Bodymovin est particulièrement adapté aux **bannières publicitaires** et aux **animations d'interface** : le motion designer livre un JSON, le développeur intègre et personnalise les textes sans aller-retour avec AE.

<span id="pratique"></span>
## Cas pratique : un générique piloté par JSON

**Objectif :** un générique de fin avec 5 noms et titres, mis à jour depuis un fichier JSON sans ouvrir la timeline.

**Structure du projet AE :**

```
Comp "GENERIQUE"
├── NOM_1      (calque texte, style grand)
├── TITRE_1    (calque texte, style petit)
├── NOM_2
├── TITRE_2
├── ...
└── NOM_5 / TITRE_5
```

Chaque paire `NOM_n / TITRE_n` est positionnée et animée une fois. Le script se charge uniquement de peupler les textes.

**Fichier `generique.json` :**

```json
[
  { "nom": "Marie Dupont",   "titre": "Directrice artistique" },
  { "nom": "Paul Martin",    "titre": "Chef opérateur" },
  { "nom": "Sofia Reyes",    "titre": "Monteuse" },
  { "nom": "Luc Bernard",    "titre": "Compositeur" },
  { "nom": "Anna Kowalski",  "titre": "Étalonnage" }
]
```

**Script `peupler-generique.jsx` :**

```javascript
const projet = app.project;
const comp = projet.activeItem;

const fichierJson = File(projet.file.parent.fsName + "/generique.json");
fichierJson.open("r");
const donnees = JSON.parse(fichierJson.read());
fichierJson.close();

app.beginUndoGroup("Peupler générique");

for (let i = 0; i < donnees.length; i++) {
    const entree = donnees[i];
    const n = i + 1;

    const calqueNom   = comp.layer("NOM_"   + n);
    const calqueTitre = comp.layer("TITRE_" + n);

    if (calqueNom)   calqueNom.property("Source Text").setValue(entree.nom);
    if (calqueTitre) calqueTitre.property("Source Text").setValue(entree.titre);
}

app.endUndoGroup();
alert("Générique mis à jour.");
```

`app.beginUndoGroup` / `app.endUndoGroup` encadrent les modifications pour qu'un seul `Ctrl+Z` annule l'ensemble de l'opération.

**Mise à jour du générique :** modifier `generique.json` dans n'importe quel éditeur texte, relancer le script dans AE. L'animation est intacte, seuls les textes changent.
