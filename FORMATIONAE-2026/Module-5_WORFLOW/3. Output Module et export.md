## La file d'attente de rendu 

La **Render Queue** (file d'attente de rendu) est le système d'export natif d'AE. Elle permet de rendre une ou plusieurs compositions, avec des paramètres précis, sans passer par un logiciel tiers.

**Ouvrir la Render Queue :** Fenêtre → File d'attente de rendu, ou `Cmd+Alt+0` (Mac) / `Ctrl+Alt+0` (Windows).

**Ajouter une composition à la file :** depuis la timeline ou le panneau Projet, `Cmd+Shift+/` (Mac) / `Ctrl+Shift+/` (Windows), ou glisser la comp dans la Render Queue.

Chaque entrée dans la file affiche deux lignes de paramètres configurables :
- **Render Settings** : qualité, framerate, résolution, durée
- **Output Module** : format, codec, canaux, profil colorimétrique
- **Output To** : chemin et nom du fichier de sortie

<span id="parametres"></span>
## Render Settings : les paramètres de rendu 

Clique sur le lien bleu **"Best Settings"** (ou le nom du preset actif) pour ouvrir les Render Settings.

**Paramètres essentiels :**

```
Quality         → Best (toujours, sauf pour des tests rapides)
Resolution      → Full (100%) pour le livrable final
                  Half ou Third pour les previews
Time Span       → Work Area Only (zone de travail) ou Length of Comp (toute la comp)
Frame Rate      → "Use comp's frame rate" dans 99% des cas
Field Render    → Off (sauf pour de la vidéo entrelacée, rare aujourd'hui)
```

> 💡 Crée un preset "Final" avec Quality=Best, Resolution=Full, Time Span=Comp et sauvegarde-le. Tu l'utiliseras pour tous tes rendus finals.

<span id="output"></span>
## Output Module : le format de sortie 

Clique sur le lien bleu du module de sortie pour choisir le format et le codec.

**Paramètres essentiels :**

```
Format          → le conteneur (QuickTime, AVI, PNG Sequence, TIFF Sequence...)
Format Options  → le codec à l'intérieur du conteneur
Channels        → RGB (sans transparence) ou RGB + Alpha (avec transparence)
Depth           → Millions of Colors+ pour le canal alpha
Color           → Premultiplied ou Straight (impacte la transparence)
```

**Stretch** permet de rendre dans une résolution différente de la composition, utile pour des exports en plusieurs résolutions depuis la même file.

<span id="formats"></span>
## Formats et cas d'usage 

| Besoin | Format recommandé | Notes |
|---|---|---|
| Livrable vidéo final (web, client) | H.264 via Media Encoder | Ne pas encoder H.264 directement depuis la Render Queue |
| Master haute qualité | ProRes 422 HQ (Mac) ou DNxHR (Win) | Fichiers lourds, qualité maximale |
| Animation avec transparence | ProRes 4444 ou PNG Sequence | Pour intégration dans d'autres logiciels |
| Séquence d'images pour VFX | EXR Sequence ou TIFF Sequence | 32 bits, HDR, pour compositing |
| GIF animé | Via Media Encoder | AE ne génère pas de GIF nativement |
| Préview rapide | QuickTime Photo-JPEG | Léger, acceptable pour les révisions |

**Pourquoi éviter H.264 directement depuis AE ?**

AE encode le H.264 de manière inefficace comparé à Media Encoder, les fichiers sont plus lourds et le processus bloque AE pendant tout le rendu. La solution : rendre un master ProRes depuis AE, puis encoder en H.264 dans Media Encoder.

## Media Encoder : exporter sans bloquer AE 

**Adobe Media Encoder** est le logiciel d'encodage de la suite Creative Cloud. Il prend en charge la plupart des formats de sortie, et surtout il **tourne en arrière-plan**, AE reste utilisable pendant l'encodage.

**Workflow recommandé :**

1. Dans AE, composer → Ajouter à la file Adobe Media Encoder (`Cmd+Alt+M` / `Ctrl+Alt+M`)
2. Media Encoder s'ouvre avec la composition dans sa file
3. Choisir le preset de sortie (H.264, ProRes, etc.)
4. Lancer l'encodage dans Media Encoder, AE est libre

**Presets utiles dans Media Encoder :**

```
H.264 — Match Source High Bitrate   → livrable web standard
H.264 — YouTube 1080p HD            → optimisé pour l'upload YouTube
ProRes 422 LT                       → le format de diff de TV Vendée
```

> 💡 Media Encoder supporte les **Watch Folders** (dossiers surveillés) — tu déposes un fichier dans un dossier, il l'encode automatiquement avec les paramètres définis. Utile pour automatiser des exports répétitifs.
