## La précomp comme module réutilisable <span id="reutilisable"></span>

Une précomposition peut être **utilisée plusieurs fois** dans des compositions différentes — comme une instance dans un logiciel de design. Chaque utilisation pointe vers la même source : modifier la précomp met à jour toutes ses instances automatiquement.

**Exemple concret :** un bouton animé utilisé 8 fois dans une interface. Sans précomp, 8 calques à maintenir. Avec une précomp "Bouton", on modifie une seule source et les 8 instances se mettent à jour.

**Dupliquer vs Instancier :**

```
Dupliquer un calque de précomp (Cmd+D) → crée une nouvelle instance
  → pointe toujours vers la même précomp
  → modifier la précomp source met à jour toutes les instances ✅

Dupliquer la précomp dans le panneau Projet
  → crée une copie indépendante
  → les deux précomps évoluent séparément
```

> 💡 Garde cette règle en tête : **duplique le calque dans la timeline** pour des instances liées, **duplique la comp dans le Projet** pour des variantes indépendantes.

**Organisation recommandée dans le panneau Projet :**

```
Projet
├── _MAIN                 ← compositions principales (livrables)
├── _PRECOMPS             ← modules réutilisables
│   ├── Bouton_anim
│   ├── Logo_reveal
│   └── Transition_wipe
└── _ASSETS               ← footages, images, sons
```

## comp() : accéder à une autre composition par expression <span id="comp"></span>

La fonction `comp()` permet de **référencer n'importe quelle composition du projet** depuis une expression, et d'en lire les propriétés de calques.

```javascript
comp("Nom de la composition")
// → retourne un objet Composition, identique à thisComp
//   mais pour une comp différente
```

**Syntaxe complète pour lire une valeur depuis une autre comp :**

```javascript
comp("Nom de la comp").layer("Nom du calque").transform.position
```

**Cas d'usage — synchroniser deux compositions :**

Tu as une comp "Données" qui contient un calque texte servant de source de données (un compteur, une valeur calculée). Une deuxième comp "Affichage" veut lire cette valeur.

```javascript
// Dans la comp "Affichage", sur sourceText d'un calque texte :
comp("Données").layer("Compteur").text.sourceText
```

**Cas d'usage — déclencher une animation depuis une comp maître :**

```javascript
// Sur la propriété Opacité d'un calque dans une précomp :
var signal = comp("MASTER").layer("Trigger").transform.opacity;
linear(signal, 0, 100, 0, 100)
// La précomp réagit à l'opacité d'un calque dans la comp maître
```

> 💡 `comp()` est puissant mais crée des **dépendances entre compositions**. Utilise-le avec parcimonie — des dépendances circulaires (comp A lit comp B qui lit comp A) peuvent créer des erreurs difficiles à déboguer.

## Master Properties <span id="master"></span>

Les Master Properties sont une fonctionnalité qui permet d'**exposer des propriétés de l'intérieur d'une précomp** vers l'extérieur — directement sur le calque de précomp dans la composition parente.

C'est la version structurée et propre du système de contrôle : au lieu de passer par `comp()` ou par des calques CONTROLS complexes, tu définis explicitement quelles propriétés sont "paramétrables" depuis l'extérieur.

**Pour créer une Master Property :**

1. Ouvre la précomp
2. Dans la timeline, `Alt+clic` sur le chronomètre d'une propriété → l'icône change
3. Retourne dans la comp parente → le calque de précomp expose maintenant cette propriété dans ses **Essential Graphics** (Propriétés essentielles)

```
Comp parente
└── Calque "Bouton_anim" (précomp)
    ├── Couleur fond     ← Master Property exposée
    ├── Texte label      ← Master Property exposée
    └── Durée animation  ← Master Property exposée
```

Chaque instance du calque de précomp peut avoir des valeurs différentes pour ces propriétés — tout en partageant la même logique d'animation interne.

> 💡 Les Master Properties sont la base des **MOGRT** (Motion Graphics Templates) utilisés dans Premiere Pro. Une précomp avec des Master Properties bien définies peut être exportée comme template réutilisable par des monteurs qui n'ont pas accès à AE.
