<span id="au-dela"></span>
# Au-delà du parent unique

Dans la page 1 de ce module, on a vu le cas de base : un calque enfant hérite des transformations d'un calque parent. C'est suffisant pour des cas simples. Mais dès qu'on construit des systèmes plus élaborés, un personnage articulé, un mécanisme, une infographie animée complexe, on enchaîne plusieurs niveaux de parenté les uns sur les autres.

Une **chaîne de parenté** est une hiérarchie de calques où chaque niveau hérite de tous les niveaux au-dessus de lui :

```
Grand-parent
└── Parent
    └── Enfant
        └── Petit-enfant
```

Chaque calque de la chaîne hérite de **l'intégralité** des transformations de tous ses ancêtres, pas seulement de son parent direct. Le petit-enfant est donc influencé par le grand-parent, le parent, et lui-même.

> 💡 Analogie : pense à un bras humain. L'épaule bouge → l'avant-bras suit. L'avant-bras tourne → la main suit. La main pivote → les doigts suivent. Chaque segment est l'enfant du précédent. Bouger l'épaule déplace tout le bras d'un coup, c'est exactement ce que fait une chaîne de parenté dans AE.

<span id="chaine-niveaux"></span>
# La chaîne à plusieurs niveaux

**Mise en place d'une chaîne à 3 niveaux :**

```
Timeline :
  NULL_Corps        ← le maître, ancêtre de tout
  └── NULL_Bras     ← enfant de Corps, parent de Main
      └── Calque_Main ← enfant de Bras
```

Pour assigner la parenté dans AE : dans la colonne **Parent & Lien** de la timeline, sélectionner le parent dans le menu déroulant de chaque calque. Si la colonne n'est pas visible : clic droit sur l'en-tête de la timeline → **Colonnes** → **Parent & Lien**.

**Ce qui se passe quand on anime NULL_Corps :**

```
NULL_Corps se déplace de [100, 0]
→ NULL_Bras se déplace de [100, 0]  (hérite)
→ Calque_Main se déplace de [100, 0] (hérite de Bras, qui hérite de Corps)
```

**Ce qui se passe quand on anime NULL_Bras :**

```
NULL_Bras pivote de 45°
→ Calque_Main pivote de 45° autour du pivot de Bras (hérite)
→ NULL_Corps n'est pas affecté (les transformations ne remontent pas)
```

Les transformations descendent dans la chaîne, elles ne remontent jamais.

<span id="heritage"></span>
# Héritage des transformations : ce qui se cumule

Toutes les transformations d'un parent se transmettent à ses enfants, mais elles ne se combinent pas toutes de la même façon.

**Position :** s'additionne à chaque niveau.

```
Parent Position : [200, 100]
Enfant Position (locale) : [50, 0]
→ Position réelle de l'enfant dans la comp : [250, 100]
```

**Rotation :** s'additionne à chaque niveau.

```
Grand-parent Rotation : 30°
Parent Rotation : 20°
Enfant Rotation (locale) : 10°
→ Rotation réelle de l'enfant dans la comp : 60°
```

**Scale :** se multiplie à chaque niveau, c'est le comportement le plus piégeux.

```
Parent Scale : 50%
Enfant Scale (locale) : 50%
→ Scale réelle de l'enfant dans la comp : 25%  (50% × 50%)
```

> 💡 Ce comportement multiplicatif de la Scale est la source d'erreur la plus fréquente dans les chaînes de parenté. Si tu veux qu'un enfant garde sa taille réelle quelle que soit la scale du parent, il faut compenser manuellement, ou utiliser une expression de compensation (voir la section dédiée plus bas).

**Opacité :** ne s'hérite PAS via la parenté. L'opacité d'un parent n'affecte pas celle de ses enfants dans la timeline. Pour un comportement d'opacité groupée, il faut utiliser une précomposition.

<span id="rotation-externe"></span>
# Rotation autour d'un point extérieur

L'un des cas d'usage les plus puissants des chaînes de parenté est la **rotation autour d'un point qui n'est pas le centre du calque**. C'est impossible avec une rotation simple, le calque pivote toujours autour de son propre point d'ancrage. La chaîne de parenté résout ce problème élégamment.

**Exemple : une planète qui tourne autour du soleil.**

```
NULL_Pivot  → positionné au centre du soleil, animé en Rotation
└── Calque_Planete → positionné à distance du pivot (ex : [300, 0] en local)
```

Quand NULL_Pivot tourne, la planète décrit une orbite circulaire autour du soleil, sans aucune keyframe sur la planète elle-même, et sans expression.

**Mise en place étape par étape :**

1. Créer un Calque Nul, le placer exactement sur le point de pivot souhaité (le centre du soleil).
2. Créer le calque enfant (la planète), le placer à la distance voulue en déplaçant sa **position locale** (pas le nul).
3. Assigner le nul comme parent de la planète.
4. Animer la **Rotation** du nul.

```
NULL_Pivot : Position [960, 540] — centre de la comp
  Rotation : 0° à 0s → 360° à 5s  (un tour en 5 secondes)

Calque_Planete : Position locale [300, 0]
  → orbite circulaire de rayon 300px autour de [960, 540]
```

Cette technique fonctionne à tous les niveaux d'une chaîne : chaque nul intermédiaire est un pivot potentiel pour ses enfants.

<span id="articulation"></span>
# Simuler une articulation : bras, roue, aiguille

Les chaînes de parenté sont la technique standard pour simuler des systèmes articulés dans AE. Voici trois configurations typiques.

## Bras articulé à deux segments

```
NULL_Epaule  → pivot de l'épaule, animé en Rotation
└── NULL_Coude  → pivot du coude, positionné à l'extrémité du bras supérieur
    └── Calque_AvanBras  → le segment visuel de l'avant-bras
```

**Clé de construction :** le point d'ancrage de chaque segment doit être placé à l'articulation, pas au centre du calque. Pour `Calque_AvanBras`, l'ancre doit être au niveau du coude (extrémité haute du segment), pas au milieu.

```
// Déplacer l'ancre d'un calque sans le faire bouger visuellement :
// Outil Point d'ancrage (Y) dans AE — déplace l'ancre indépendamment du calque
// Ou : Layer → Transform → Center Anchor Point in Layer Content (ne convient pas toujours)
```

## Roue avec rayons

```
NULL_Centre_Roue  → animé en Rotation (la roue tourne)
├── Calque_Rayon_1  → position locale [0, -80]  (pointe vers le haut)
├── Calque_Rayon_2  → position locale [80, 0]   (pointe vers la droite)
└── Calque_Rayon_3  → position locale [-80, 0]  (pointe vers la gauche)
```

Un seul Animator sur le nul central fait tourner tous les rayons ensemble.

## Aiguille de compteur

```
NULL_Pivot_Aiguille → positionné à la base de l'aiguille, animé en Rotation
└── Calque_Aiguille → ancre placée à sa base (extrémité vers le pivot)
```

L'aiguille pivote autour de sa base, comme une vraie aiguille de montre.

> 💡 Dans tous ces cas, la règle est la même : **le nul va là où est l'articulation, l'ancre du calque visuel va là où est l'articulation**. Ces deux points doivent coïncider pour que la rotation soit correcte.

<span id="compensation"></span>
# Détacher un enfant sans le faire bouger : expression de compensation

Situation courante : on veut qu'un calque enfant **hérite de la position** de son parent, mais **pas de sa rotation** (ou de sa scale). La parenté standard transmet tout ou rien, il faut passer par une expression pour un héritage sélectif.

## Annuler la rotation héritée

Sur la propriété **Rotation** du calque enfant :

```javascript
// Annuler la rotation du parent pour rester droit dans la composition :
const parentRot = thisLayer.parent.transform.rotation;
-parentRot
// → si le parent pivote de 30°, l'enfant contre-tourne de -30° → reste vertical
```

Pour remonter plusieurs niveaux (grand-parent inclus) :

```javascript
const p1 = thisLayer.parent.transform.rotation;
const p2 = thisLayer.parent.parent.transform.rotation;
-(p1 + p2)
```

## Annuler la scale héritée

Sur la propriété **Scale** du calque enfant :

```javascript
// Annuler la scale du parent pour garder sa taille réelle :
const parentScale = thisLayer.parent.transform.scale / 100;
// parentScale est un Array [x, y] entre 0 et 1 (100% → 1)
[value[0] / parentScale[0], value[1] / parentScale[1]]
// → si le parent est à 50%, l'enfant se compense à 200% → taille réelle maintenue
```

## Suivre la position sans hériter de la rotation : le cas du texte d'étiquette

Un cas très fréquent en motion design : une étiquette de texte qui doit rester lisible (horizontale) tout en suivant un élément qui tourne.

```javascript
// Sur la Rotation de l'étiquette, enfant d'un calque qui tourne :
-thisLayer.parent.transform.rotation

// Sur la Position de l'étiquette : parenté normale — elle suit le parent
// Sur la Rotation : expression de compensation — elle reste droite
```

> 💡 Cette technique, héritage de position sans héritage de rotation, est tellement courante qu'elle mérite d'être mémorisée. C'est la recette de tout élément "ancré dans l'espace" qui reste lisible quelle que soit l'orientation de son support.

<span id="pieges"></span>
# Pièges courants et comment les éviter

## Assigner un parent après avoir positionné un calque

Quand on assigne un parent à un calque déjà positionné, AE **recalcule la position locale** pour que le calque reste visuellement au même endroit. La valeur affichée dans la timeline change, mais le calque ne bouge pas à l'écran. C'est le comportement attendu et correct.

En revanche, si on change le parent d'un calque qui a déjà des keyframes de position, les keyframes deviennent incorrectes, elles étaient en local space de l'ancien parent, pas du nouveau.

```
// Bonne pratique : assigner les parents AVANT de créer les keyframes
// Si ce n'est pas possible : utiliser Edit → Paste Special pour recoller
// les keyframes dans le bon référentiel
```

## Scale non uniforme qui déforme les enfants

Si un parent a une scale non uniforme, par exemple `[200%, 50%]` les enfants sont déformés dans le même sens, et leurs rotations semblent incorrectes (les angles sont faussés par la déformation). C'est une limitation d'AE : les chaînes de parenté avec scale non uniforme ne fonctionnent pas comme en 3D.

```
// Solution : éviter les scales non uniformes sur les calques qui ont des enfants
// Si nécessaire, appliquer la déformation via un effet (Distorsion) plutôt que via la Scale
```

## Perdre le fil de la hiérarchie sur un projet complexe

Avec 6 ou 7 niveaux de parenté, il devient difficile de comprendre qui dépend de qui.

```
// Bonnes pratiques d'organisation :
// 1. Nommer tous les nulls avec un préfixe clair : NULL_, PIVOT_, CTRL_
// 2. Utiliser les couleurs d'étiquettes pour regrouper visuellement les chaînes
// 3. Utiliser les calques de guide (Layer → Guide Layer) pour les nulls non rendus
// 4. Indenter les calques dans la timeline en respectant l'ordre hiérarchique
//    (l'enfant en dessous du parent dans la liste)
```

## Supprimer un maillon intermédiaire de la chaîne

Supprimer un nul intermédiaire casse la chaîne, les enfants perdent leur parent et restent à leur position locale sans référence.

```
// Avant de supprimer un nul intermédiaire :
// 1. Réassigner les enfants au nul du niveau supérieur
// 2. Vérifier visuellement que rien n'a bougé
// 3. Supprimer le nul devenu inutile
```
