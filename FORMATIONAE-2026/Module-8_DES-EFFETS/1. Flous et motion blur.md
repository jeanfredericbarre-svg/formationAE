## Vue d'ensemble des différents flous <span id="lesflous"></span>

AE propose plusieurs effets de flou très différents dans leur usage et leurs performances. Le choix dépend du résultat visuel recherché, pas de la polyvalence.

| Effet                    | GPU | Usage principal                              |
| ------------------------ | --- | -------------------------------------------- |
| **Gaussian Blur** | ❌ | Remplacé par Fast Box Blur — à éviter en production |
| **Fast Box Blur**        | ✅   | Flou général — couvre 90 % des besoins       |
| **Directional Blur**     | ❌   | Vitesse, dynamisme, entrée dans le cadre     |
| **Camera Lens Blur**     | ❌   | Bokeh réaliste, profondeur de champ          |
| **Radial Blur**          | ❌   | Rotation, zoom rapide                        |
| **Motion Blur natif**    | ✅   | Calques animés par keyframes ou expressions  |
| **CC Force Motion Blur** | ❌   | Motion blur post-render sur footage existant |

> **Règle de production :** Fast Box Blur à 3 itérations couvre 90 % des besoins. Les autres sont des outils spécialisés — on les choisit pour leur effet visuel.

## Fast Box Blur <span id="fastboxblur"></span>

**Effect → Blur & Sharpen → Fast Box Blur**

Remplaçant direct du Gaussian Blur pour la production. Accéléré GPU, il produit un résultat visuellement très proche à 3–5× la vitesse. À rayon équivalent, quasi-indiscernable d'un Gaussian à partir de 3 itérations.

```
Blur Radius        → rayon du flou en pixels
Iterations         → nombre de passes
                     1 = carré visible aux bords
                     3 = approxime un gaussian correct
                     5+ = quasi-indiscernable d'un gaussian
Blur Dimensions    → Horizontal and Vertical / Horizontal / Vertical
Repeat Edge Pixels → évite le halo sombre aux bords du calque
                     à activer systématiquement sur les footages non-transparents
```

> **Repeat Edge Pixels** est le paramètre le plus oublié. Sans lui, les bords du calque sont floutés vers du transparent — ce qui crée un liseré sombre disgracieux sur fond coloré.

Expression pour un flou dynamique lié à la vitesse du calque (sur le paramètre Blur Radius) :

```javascript
var vitesse = thisLayer.transform.position.speed;
linear(vitesse, 0, 500, 0, 40)
// Plus le calque bouge vite, plus le flou est fort
```

## Directional Blur <span id="directional"></span>

**Effect → Blur & Sharpen → Directional Blur**

Applique un flou dans une direction précise, simulant le flou de mouvement d'un objet qui se déplace vite dans le cadre.

```
Direction     → angle du flou (0° = horizontal, 90° = vertical)
Blur Length   → intensité du flou en pixels
```

Usage typique — objet entrant dans le cadre à grande vitesse :

```
Frame 0   : Blur Length = 60, Opacity = 0
Frame 10  : Blur Length = 0,  Opacity = 100
```

> Combine Directional Blur avec Motion Blur natif pour un résultat encore plus convaincant. Les deux s'additionnent.

**Variation — zoom blur :** pour simuler un zoom rapide, utilise **Radial Blur** (Effect → Blur & Sharpen → Radial Blur) en mode Zoom, avec le point d'origine centré sur le sujet.

## Camera Lens Blur <span id="lensblur"></span>

**Effect → Blur & Sharpen → Camera Lens Blur**

Le flou le plus réaliste d'AE — simule réellement une optique photographique avec forme de l'iris, aberrations chromatiques et highlights en bokeh. C'est aussi le plus lent (rendu CPU pur). À réserver aux plans importants, pas aux backgrounds.

Travailler en **Quarter resolution** pendant la composition ; basculer en Full uniquement au rendu.

### Paramètres essentiels

|Paramètre|Rôle|
|---|---|
|**Blur Radius**|Rayon du flou (équivalent à l'ouverture f/)|
|**Iris Shape**|Forme de l'iris (Triangle → Décagone) — influence la forme du bokeh|
|**Blade Curvature**|0 = formes anguleuses / 100 = cercle parfait|
|**Highlight Gain**|Intensité des highlights — le paramètre "magie" ; à monter doucement|
|**Highlight Threshold**|Seuil à partir duquel les highlights s'ouvrent|
|**Chromatic Aberration**|Frange chromatique rouge/cyan aux bords du bokeh|
|**Diffraction Fringe**|Aura lumineuse aux bords des highlights|
|**Noise**|Grain ajouté dans le flou (aspect naturel)|

### Technique — profondeur de champ par Blur Map

Camera Lens Blur peut lire une couche en niveaux de gris qui pilote l'intensité du flou pixel par pixel (zones blanches = très flou, zones noires = net) :

1. Créer un Shape Layer en dégradé (noir → blanc) dans une précomp
2. Dans Camera Lens Blur → **Blur Map** → sélectionner la précomp
3. Activer **If Layer Sizes Differ → Stretch Map to Fit**

Le résultat simule une profondeur de champ avec netteté progressive, sans passer par une caméra 3D.

## Motion Blur natif <span id="motionblur"></span>

Le motion blur natif n'est pas un effet — c'est une option de rendu intégrée à AE. Il calcule le flou de mouvement réel des calques animés en fonction de leur vitesse à chaque frame.

**Activation à deux niveaux obligatoires :**

```
1. Au niveau du calque  → icône Motion Blur (●) dans les switches de la timeline
2. Au niveau de la comp → bouton Enable Motion Blur dans la barre de la timeline
```

Les deux doivent être actifs pour que le motion blur soit visible à la prévisualisation.

**Réglage de la qualité** via Composition → Composition Settings → Advanced :

```
Shutter Angle     → durée d'exposition simulée en degrés
                    90°  = très net (exposition courte)
                    180° = standard cinéma (flou modéré)
                    360° = très flou (exposition longue)
Shutter Phase     → décalage temporel du flou par rapport à la frame
                    -90° = standard (flou centré sur le mouvement)
Samples per Frame → sous-frames calculées
                    16 = qualité finale correcte
                    64 = qualité maximale, très lent
```

> Le Motion Blur natif fonctionne sur tous les calques animés — keyframes, expressions, ou via un parent. C'est toujours le premier choix pour les animations créées dans AE.

## CC Force Motion Blur <span id="ccforcemotionblur"></span>

**Effect → Time → CC Force Motion Blur**

Quand un calque n'a pas de motion blur natif — footage réel sans flou de caméra, rendu 3D, calque de synthèse — CC Force Motion Blur l'ajoute après coup.

```
Motion Blur Samples → sous-frames analysées
                      8  = bon compromis qualité/vitesse
                      16+ = usage ponctuel, qualité améliorée
Shutter Angle       → même logique que le motion blur natif
```

> **Limite :** CC Force Motion Blur analyse le mouvement du calque entier, pas le mouvement interne. Si le footage contient un objet qui bouge dans un cadre fixe (ex. une voiture dans un plan statique), le flou ne s'appliquera pas à l'objet.

## Flou en précomp <span id="flouprecomp"></span>

Quand un calque flou est statique ou en boucle, recalculer le flou à chaque frame est du gaspillage. La solution : pre-render la précomp.

**Procédure :**

1. Placer le(s) calque(s) à flouter dans une **précomp**
2. Appliquer le flou à l'intérieur de la précomp
3. Clic droit sur la précomp dans la timeline → **Pre-render**
4. AE ajoute la précomp à la Render Queue et la rend en ProRes ou PNG Sequence
5. Le fichier rendu est automatiquement réimporté comme proxy de la précomp

AE lit maintenant un fichier vidéo déjà rendu au lieu de recalculer le flou à chaque frame.

> Exemple de gain : si le flou coûte 2 s/frame sur une précomp de 5 s à 25 fps → 250 secondes économisées à chaque prévisualisation.
