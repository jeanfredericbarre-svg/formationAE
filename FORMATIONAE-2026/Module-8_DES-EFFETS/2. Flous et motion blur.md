# Choisir son flou — la carte rapide

L'apprenant qui connaît déjà Gaussian Blur se heurte rapidement à un problème : il est lent sur de grands rayons, et il ne fait qu'un seul type de flou. AE propose une palette d'effets de flou très différents dans leur usage.

| Effet | GPU | Usage principal |
|---|---|---|
| **Fast Box Blur** | ✅ | Flou général, performances |
| **Directional Blur** | ❌ | Vitesse, dynamisme |
| **Camera Lens Blur** | ❌ | Bokeh réaliste, profondeur de champ |
| **Radial Blur** | ❌ | Rotation, zoom rapide |
| **CC Force Motion Blur** | ❌ | Motion blur post-render |
| **Motion Blur natif** | ✅ | Calques animés par keyframes ou expressions |

> 💡 Règle de production : Fast Box Blur à 3 itérations couvre 90% des besoins. Les autres sont des outils spécialisés — on les choisit pour leur effet visuel, pas pour leur polyvalence.

# Fast Box Blur — le flou de production

**Effect → Blur & Sharpen → Fast Box Blur**

Fast Box Blur est le remplaçant direct de Gaussian Blur pour la production. Il est accéléré GPU et produit un résultat visuellement très proche à grande vitesse.

**Les paramètres :**

```
Blur Radius   → rayon du flou en pixels
Iterations    → nombre de passes successives
                1 = carré visible aux bords
                3 = approxime un gaussian correct
                5+ = quasi-indiscernable d'un gaussian
Blur Dimensions → Horizontal and Vertical / Horizontal / Vertical
                  → permet un flou directionnel simple
Repeat Edge Pixels → évite le halo sombre aux bords du calque
                     à activer systématiquement sur les footages non-transparents
```

**Repeat Edge Pixels** est le paramètre le plus oublié. Sans lui, les bords du calque sont floutés vers du transparent — ce qui crée un liseré sombre disgracieux sur fond coloré. Activer cette option étend les pixels de bord au lieu de flouter vers le vide.

```javascript
// Combo Fast Box Blur + expression pour un flou dynamique :
// Sur le paramètre Blur Radius, expression :
var vitesse = thisLayer.transform.position.speed;
linear(vitesse, 0, 500, 0, 40)
// Plus le calque bouge vite, plus le flou est fort
```

# Directional Blur — le flou cinétique

**Effect → Blur & Sharpen → Directional Blur**

Directional Blur applique un flou dans une **direction précise**, simulant le flou de mouvement d'un objet qui se déplace vite dans le cadre.

**Les paramètres :**

```
Direction     → angle du flou (0° = horizontal, 90° = vertical)
Blur Length   → intensité du flou en pixels
```

**Usage typique — un objet qui entre dans le cadre à grande vitesse :**

1. Ajouter Directional Blur sur le calque
2. Régler Direction dans l'axe de déplacement de l'objet
3. Animer Blur Length de 80 → 0 au moment où l'objet s'arrête

```
Frame 0   : Blur Length = 60, Opacity = 0
Frame 10  : Blur Length = 0,  Opacity = 100
```

> 💡 Combine Directional Blur avec Motion Blur natif pour un résultat encore plus convaincant. Les deux s'additionnent.

**Variation — le zoom blur :** Pour simuler un zoom rapide, utilise **Radial Blur** (Effect → Blur & Sharpen → Radial Blur) en mode Zoom. Centre le point d'origine sur le sujet de l'image.

# Lens Blur — le bokeh réaliste

**Effect → Blur & Sharpen → Camera Lens Blur**

Camera Lens Blur est le flou le plus réaliste d'AE. Il simule réellement une optique photographique — avec la forme de l'iris, les aberrations chromatiques, et le comportement des highlights qui s'ouvrent en cercles lumineux (bokeh).

**C'est aussi le plus lent.** À réserver aux plans importants, pas aux backgrounds.

**Les paramètres essentiels :**

```
Blur Radius         → rayon du flou (équivalent à l'ouverture f/)
Iris Shape          → forme de l'iris (Triangle, Carré, Pentagone... Décagone)
                      → influence directement la forme du bokeh
Blade Curvature     → arrondi des lames de l'iris
                      0 = formes anguleuses / 100 = cercle parfait
Rotation            → rotation de l'iris
Diffraction Fringe  → aura lumineuse aux bords des highlights
Highlight Gain      → intensité des highlights — le paramètre "magie"
                      → à monter doucement, il sature les zones lumineuses
Highlight Threshold → seuil à partir duquel les highlights s'ouvrent
Chromatic Aberration → frange chromatique (rouge/cyan) aux bords du bokeh
```

**Technique — profondeur de champ pilotée par un plan de flou :**

Camera Lens Blur peut lire un **Blur Map** — une couche en niveaux de gris qui pilote l'intensité du flou pixel par pixel. Les zones blanches sont très floues, les zones noires sont nettes.

1. Créer un Shape Layer en dégradé (noir → blanc) en précomp
2. Dans Camera Lens Blur → **Blur Map** → sélectionner la précomp
3. Activer **If Layer Sizes Differ → Stretch Map to Fit**

Le résultat simule une profondeur de champ avec une netteté progressive — sans passer par une caméra 3D.

> 💡 Pour les flous de fond rapides (background flou derrière un sujet net), préfère toujours Fast Box Blur. Camera Lens Blur est pour les situations où la qualité du bokeh est visible et importante.

# Motion Blur natif d'AE

Le motion blur natif n'est pas un effet — c'est une option de rendu intégrée à AE. Il calcule le flou de mouvement réel des calques animés, en fonction de leur vitesse à chaque frame.

**Activation à deux niveaux :**

```
1. Au niveau du calque    → icône Motion Blur (●) dans les switches de la timeline
2. Au niveau de la comp   → bouton Enable Motion Blur dans la barre de la timeline
```

Les deux doivent être actifs pour que le motion blur soit calculé et visible à la prévisualisation.

**Régler la qualité du motion blur :**

`Composition → Composition Settings → Advanced` → section **Motion Blur** :

```
Shutter Angle   → durée d'exposition simulée en degrés
                  180° = standard cinéma (flou modéré)
                  360° = très flou (exposition longue)
                  90°  = très net (exposition courte)
Shutter Phase   → décalage temporel du flou par rapport à la frame
                  -90° = standard (flou centré sur le mouvement)
Samples per Frame → nombre de sous-frames calculées
                    16 = qualité finale correcte
                    64 = qualité maximale, très lent
```

> 💡 Le Motion Blur natif fonctionne sur tous les calques animés — qu'ils bougent par keyframes, par expressions, ou via un parent. C'est toujours le premier choix pour les animations créées dans AE.

# CC Force Motion Blur — le motion blur post-render

**Effect → Time → CC Force Motion Blur**

Quand un calque n'a pas de motion blur natif — footage réel sans flou de caméra, rendu 3D sans motion blur, calque synthèse — CC Force Motion Blur l'ajoute après coup.

```
Motion Blur Samples → nombre de sub-frames
                      8 = bon compromis qualité/vitesse
                      16+ = qualité améliorée, usage ponctuel
Shutter Angle       → même logique que le motion blur natif
```

**Limites :** CC Force Motion Blur analyse le mouvement du calque entre les frames — pas le mouvement interne. Si le footage contient un objet qui bouge dans le cadre (une voiture dans une prise de vue fixe), le flou ne sera pas appliqué à la voiture, seulement au calque entier.

# Flou en précomp — la technique de performance

Quand un calque flou est statique ou en boucle, recalculer le flou à chaque frame est du gaspillage pur. La solution : **pre-render la précomp**.

**Procédure :**

1. Placer le calque (ou les calques) à flouter dans une **précomp**
2. Appliquer le flou à l'intérieur de la précomp
3. Clic droit sur la précomp dans la timeline → **Pre-render**
4. AE ajoute la précomp à la Render Queue et la rend en ProRes ou PNG Sequence
5. Le fichier rendu est automatiquement réimporté comme proxy de la précomp

AE lit maintenant un fichier vidéo déjà rendu au lieu de recalculer le flou à chaque frame.

> 💡 Gain typique : si le flou coûtait 2 secondes par frame et que la précomp dure 5 secondes à 25fps, c'est 250 secondes de calcul économisées à chaque prévisualisation.
