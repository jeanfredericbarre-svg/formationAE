## CC Composite : recompositer sans dupliquer 

**Effect → Channel → CC Composite**

CC Composite est l'un des effets les plus discrets et les plus utiles d'AE. Son rôle : **recompositer le calque original en dessous des effets** qui lui ont été appliqués, en un seul calque, sans avoir à dupliquer.

**Le problème qu'il résout :**

Tu appliques un effet de flou ou de glow sur un calque — mais tu veux que l'original reste visible en dessous, net, et que le flou s'ajoute par-dessus en mode Screen. Sans CC Composite, la solution classique est de dupliquer le calque :

```
Timeline classique (sans CC Composite) :
├── Calque A  [Fast Box Blur, Mode Screen]   ← copie floue
└── Calque A  [original]                     ← original net
```

Avec CC Composite, tout tient en un seul calque :

```
Timeline avec CC Composite :
└── Calque A
    ├── Fast Box Blur (rayon 20)
    └── CC Composite     ← recolle le calque original EN DESSOUS du flou
```

**Les paramètres :**

```
Composite Original → mode de fusion avec lequel l'original est recomposité
                     In Front / Behind / Blend / Screen / Multiply / Add...
Opacity            → opacité de l'original dans le recompositage
RGB Only           → ignore le canal alpha de l'original
```

**Le paramètre clé est `Composite Original`** — il définit comment l'original se mélange avec le résultat des effets appliqués au-dessus de lui.

**Usage typique — glow lumineux sur un texte :**

```
Calque texte
├── Fast Box Blur (rayon 25)
└── CC Composite (Composite Original = In Front)
```

Le texte flou (le glow) est en dessous, le texte net est par-dessus — en un seul calque. Modifier le texte met à jour les deux simultanément.

**Usage typique — effet de double exposition :**

```
Calque footage
├── Hue/Saturation (désaturation totale → image en N&B)
├── Curves (fort contraste)
└── CC Composite (Composite Original = Screen, Opacity = 60%)
```

L'original en couleur se recomposite en Screen sur la version N&B contrastée — double exposition en un seul effet.

> 💡 CC Composite est particulièrement puissant combiné avec des effets de distorsion. Appliquer Turbulent Displace + CC Composite (In Front) maintient la version originale nette superposée à la version distordue — sans calque supplémentaire.

## Set Matte : le Track Matte sans contrainte d'ordre 

**Effect → Channel → Set Matte**

Set Matte remplit la même fonction que le Track Matte — utiliser un calque comme masque d'un autre — mais sans la contrainte d'ordre dans la timeline. Le calque matte peut être **n'importe où** dans la composition.

**Le problème que résout Set Matte :**

Le Track Matte impose que le calque matte soit **directement au-dessus** du calque cible. Dès qu'on travaille avec des précomps, des calques d'ajustement, ou une timeline complexe, cette contrainte devient vite un casse-tête.

**Les paramètres :**

```
Take Matte From Layer → sélectionner n'importe quel calque de la composition
Use For Matte         → quel canal du calque matte est utilisé
                        Alpha Channel / Red / Green / Blue / Luminance / Hue
Invert Matte          → inverse la transparence
Stretch Matte to Fit  → adapte la taille du matte à celle du calque cible
Composite Matte with Original → combine le matte avec l'alpha existant
                                 au lieu de le remplacer
```

**Usage typique — matte depuis un calque lointain dans la timeline :**

```
Timeline :
├── Calque d'ajustement grading
├── Texte principal
├── Logo SVG animé     ← ce calque sert de matte
├── Calque d'ajustement intermédiaire
└── Vidéo              ← Set Matte → Take Matte From Layer = "Logo SVG animé"
                          La vidéo n'apparaît que dans la forme du logo
```

Impossible à faire proprement avec Track Matte sans réorganiser toute la timeline.

**Usage typique — matte partagé entre plusieurs calques :**

Un seul calque matte peut être référencé par **autant de calques que nécessaire** via Set Matte — chaque calque a son propre effet Set Matte pointant vers le même source. Avec Track Matte, chaque calque cible nécessiterait sa propre copie du matte.

> 💡 Set Matte et Track Matte ne s'excluent pas. Dans une composition complexe, il est courant d'utiliser Track Matte pour les paires simples (où l'ordre n'est pas contraignant) et Set Matte pour les situations où la timeline ne peut pas être réorganisée.

## Compound Arithmetic : opérations entre deux calques 

**Effect → Channel → Compound Arithmetic**

Compound Arithmetic applique une **opération mathématique** entre les pixels du calque courant et les pixels d'un autre calque de la composition. C'est une version programmatique et précise des modes de fusion — avec accès à des opérations qui n'existent pas en blend mode.

**Les paramètres :**

```
Second Source Layer → le calque qui sert d'opérande
Operator           → l'opération appliquée pixel par pixel
                     Add / Subtract / Multiply / Divide / Min / Max /
                     And / Or / Xor / Difference / Exclusion /
                     If Source > Second Source / If Source < Second Source
Channel Operator   → s'applique sur tous les canaux ou séparément
Clamp Result       → plafonne le résultat entre 0 et 1 (évite les valeurs extrêmes)
```

**Les opérateurs les plus utiles :**

**Min** — conserve la valeur la plus sombre des deux calques pixel par pixel. Résultat similaire à Multiply mais sans l'atténuation progressive — plus brutal, plus contrasté.

**Max** — conserve la valeur la plus claire des deux calques. Résultat similaire à Screen mais plafonné.

**Xor** — opération bit-à-bit. Sur des images en niveaux de gris, produit des inversions partielles complexes — utile pour des effets de glitch ou des motifs abstraits.

**If Source > Second Source** — conserve les pixels du calque courant seulement là où il est plus clair que le second calque. Tous les autres pixels passent à noir. Crée des effets de seuillage très nets.

**Usage typique — extraire les zones de lumière d'un calque :**

```
Calque A : footage
Calque B : copie de A avec Levels poussé (beaucoup plus clair)

Sur Calque A :
Compound Arithmetic
  Second Source = Calque B
  Operator      = Subtract
→ Résultat : seules les zones qui ont changé entre A et B sont visibles
→ Extraction des highlights avec un comportement différent de Curves
```

> 💡 Compound Arithmetic est rarement utilisé seul — c'est souvent le dernier maillon d'une chaîne d'effets, pour affiner un résultat qui ne peut pas être obtenu autrement avec les blend modes standard.

## Shift Channels : réassigner les canaux RGBA 

**Effect → Channel → Shift Channels**

Shift Channels est un outil de **réassignation des canaux de couleur**. Il permet de faire pointer chaque canal (R, G, B, A) vers n'importe quel autre canal du calque — ou vers une valeur fixe (Full On ou Off).

**Les paramètres :**

```
Take Red from   → quel canal source alimente le canal Rouge de sortie
Take Green from → idem pour le Vert
Take Blue from  → idem pour le Bleu
Take Alpha from → idem pour le canal Alpha

Valeurs disponibles pour chaque canal :
Red / Green / Blue / Alpha / Luminance / Full On / Off
```

**Usages concrets :**

**Convertir en niveaux de gris par un canal unique :**

Au lieu d'utiliser Hue/Saturation pour désaturer, on peut extraire un canal spécifique qui donne un résultat différent :

```
Take Red from   = Luminance
Take Green from = Luminance
Take Blue from  = Luminance
→ Résultat identique à une désaturation standard

Take Red from   = Red
Take Green from = Red
Take Blue from  = Red
→ N&B basé uniquement sur le canal rouge — luminosité différente
  (les ciels deviennent très clairs, la végétation très sombre)
```

**Utiliser un canal de couleur comme alpha :**

```
Take Alpha from = Luminance
→ Les zones blanches du calque deviennent opaques,
  les zones noires deviennent transparentes
→ Crée un alpha à partir de la luminance — similaire à Luma Matte,
  mais appliqué directement sur le calque sans calque matte séparé
```

**Inverser uniquement l'alpha :**

```
Take Alpha from = Alpha (avec Invert activé dans un Invert effect)
```

Ou plus directement : utiliser Shift Channels pour déplacer l'alpha vers un autre canal, traiter ce canal, puis le ramener.

**Effet chromatic aberration manuel :**

Combiner Shift Channels avec un léger Transform (déplacement de quelques pixels) sur un canal séparé produit une aberration chromatique artisanale :

```
Calque A (rouge isolé)   : Shift Channels (R=Red, G=Off, B=Off) + Transform (+2px horizontal)
Calque B (vert original) : Shift Channels (R=Off, G=Green, B=Off)
Calque C (bleu isolé)    : Shift Channels (R=Off, G=Off, B=Blue) + Transform (-2px horizontal)
→ Les trois calques en Add → frange rouge/bleue aux bords des contrastes
```

> 💡 Shift Channels est particulièrement utile quand on reçoit des rendus 3D multi-pass (passes de profondeur, passes d'occlusion, passes de normales) où l'information utile est encodée dans un canal de couleur spécifique qu'on veut extraire.

## Grow Bounds : étendre la zone de rendu 

**Effect → Channel → Grow Bounds**

Grow Bounds est un effet utilitaire presque invisible — il n'a aucun effet visuel direct — mais il résout un problème fréquent et frustrant : **les effets de flou ou de glow qui se font couper aux bords du calque**.

**Le problème :**

Un calque AE est rendu dans les limites de sa bounding box — le rectangle qui contient ses pixels. Un Fast Box Blur de rayon 50 appliqué à un texte va flouter les pixels jusqu'à 50px au-delà du texte — mais si le texte est proche du bord de la composition, la bounding box ne s'étend pas : le flou est tronqué net.

**La solution :**

```
Grow Bounds
  Pixels → nombre de pixels d'extension de la bounding box
            à régler à une valeur supérieure ou égale au rayon du flou
```

**Placement dans la pile d'effets :**

Grow Bounds doit être placé **avant** l'effet qui déborde, dans la pile :

```
Calque texte
├── Grow Bounds (Pixels = 60)    ← en premier — étend la zone avant le flou
└── Fast Box Blur (Rayon = 50)   ← flou appliqué sur la zone étendue
```

Sans Grow Bounds, le flou est coupé. Avec, il se propage proprement dans l'espace étendu.

**Valeur recommandée :** toujours mettre Pixels légèrement supérieur au rayon du flou (rayon 50 → Grow Bounds 60). La marge évite les artefacts aux bords de l'extension.

> 💡 Grow Bounds est aussi utile avec les effets de glow tiers (Optical Flares, Glow de Video Copilot) qui débordent souvent du calque source. Un Grow Bounds en tête de pile règle le problème systématiquement.

## Minimax : éroder et dilater un alpha 

**Effect → Channel → Minimax**

Minimax agrandit ou réduit les zones opaques d'un calque de manière procédurale — une opération appelée **dilatation** (agrandir) et **érosion** (réduire) en traitement d'image.

**Les paramètres :**

```
Operation  → Minimum (érosion — réduit les zones opaques)
             Maximum (dilatation — agrandit les zones opaques)
Radius     → intensité de l'opération en pixels
Channel    → sur quel canal s'applique l'opération
             Color and Alpha / Color / Alpha / Red / Green / Blue
Direction  → Horizontal and Vertical / Horizontal / Vertical
```

**Minimum (érosion) :**

Réduit les zones opaques. Sur un calque de texte avec canal alpha, Minimum "grignote" les bords — le texte devient plus fin. Sur des niveaux de gris, assombrit l'image en propageant les valeurs sombres.

**Maximum (dilatation) :**

Agrandit les zones opaques. Sur un texte, Maximum "gonfle" les bords — le texte devient plus gras. Sur des niveaux de gris, éclaircit en propageant les valeurs claires.

**Usage typique — créer un contour offset depuis un alpha :**

```
Calque texte original
Calque texte dupliqué
├── Minimax (Operation = Maximum, Radius = 5, Channel = Alpha)
│   → le calque gonflé de 5 pixels
└── [placé en dessous de l'original]
→ L'original net + la version gonflée en dessous = contour de 5px autour du texte
```

**Usage typique — réparer un alpha mal keye :**

Un keying (extraction de fond vert) produit souvent des alpha avec des trous (pixels transparents dans des zones qui devraient être opaques) ou des fringes (pixels semi-transparents aux bords). Minimax en mode Maximum avec un petit rayon (1–3px) comble les petits trous et densifie les bords.

**Usage typique — générer un masque de lueur :**

```
Calque logo
├── Minimax (Maximum, Radius = 20, Channel = Alpha)
├── Fast Box Blur (Rayon = 15)
└── [Mode Screen, sous le logo original]
→ Halo lumineux autour du logo sans dupliquer et flouter manuellement
```

## Simple Choker / Matte Choker : raffiner les bords d'un alpha 

**Effect → Matte → Simple Choker** **Effect → Matte → Matte Choker**

Ces deux effets affinent les bords d'un canal alpha — ils "serrent" ou "desserrent" la matte pour éliminer les fringes, les halos semi-transparents, et les artefacts de keying.

### Simple Choker

Le plus rapide et le plus direct des deux.

```
Choke Matte → valeur positive = réduit la matte (serre vers l'intérieur)
              valeur négative = agrandit la matte (desserres vers l'extérieur)
```

Simple Choker est l'outil de premier recours pour ajuster rapidement un alpha trop large ou trop étroit. Valeur typique : entre -5 et +5 pour des corrections fines.

### Matte Choker

Version avancée avec un contrôle par étapes successives — deux passes de serrage/desserrage indépendantes.

```
Geometric Softness 1  → douceur de la première passe
Choke 1               → serrage de la première passe
                        positif = serre / négatif = desserre
Gray Level Softness 1 → affecte les pixels semi-transparents (anti-alias)
Geometric Softness 2  → douceur de la deuxième passe
Choke 2               → serrage de la deuxième passe
Gray Level Softness 2 → idem pour la deuxième passe
```

**La logique des deux passes :**

La puissance de Matte Choker vient de la combinaison des deux passes en opposition. La technique classique pour éliminer une frange lumineuse (halo clair autour d'un sujet keye) :

```
Choke 1 = +20   → serre fortement pour couper la frange
Choke 2 = -18   → desserre presque autant pour récupérer les détails fins
                  (cheveux, bords translucides)
Gray Level Softness 1 = 80  → garde les demi-tons aux bords
```

Résultat : la frange lumineuse est éliminée, mais les bords fins et l'antialiasing sont préservés — ce qu'un Simple Choker seul ne peut pas faire.

**Usage typique — pipeline de keying complet :**

```
Calque footage fond vert
├── Keylight (ou autre keyer)       → extraction initiale
├── Simple Choker (Choke = +2)      → serrage global rapide
└── Matte Choker (Choke1=+15, Choke2=-12, Gray Softness=70)
                                     → raffinement des bords
```

## Echo : la traîne temporelle 

**Effect → Time → Echo**

Echo superpose **plusieurs frames du passé (ou du futur)** d'un calque sur la frame courante. Le résultat : une traîne fantôme qui suit le mouvement, ou des effets stroboscopiques.

**Les paramètres :**

```
Echo Time (seconds) → décalage temporel entre chaque écho
                      négatif = frames du passé (traîne derrière le mouvement)
                      positif = frames du futur (traîne devant le mouvement)
Number of Echoes    → combien d'échos sont superposés
Starting Intensity  → opacité du premier écho (le plus récent)
Decay               → facteur de diminution de l'opacité à chaque écho
                      1.0 = tous les échos ont la même opacité
                      0.5 = chaque écho est deux fois plus transparent que le précédent
Echo Operator       → comment les échos se combinent entre eux
                      Add / Maximum / Minimum / Screen / Composite in Back /
                      Composite in Front / Blend
```

**Echo Time, Number of Echoes et Decay forment un triangle :**

```
Echo Time = -0.05  → 3 centièmes de seconde entre chaque écho (traîne courte et serrée)
Number of Echoes = 10
Decay = 0.7        → chaque écho vaut 70% de l'écho précédent → disparition progressive
```

**Usage typique — traîne de mouvement photographique :**

```
Echo Time         = -0.04
Number of Echoes  = 8
Starting Intensity = 1.0
Decay             = 0.6
Echo Operator     = Maximum
```

Maximum conserve la valeur la plus claire à chaque pixel superposé — résultat proche d'une longue exposition photographique.

**Usage typique — stroboscope :**

```
Echo Time         = -0.1   (1/10e de seconde)
Number of Echoes  = 5
Starting Intensity = 0.8
Decay             = 1.0    (tous les échos à la même opacité — pas de fondu)
Echo Operator     = Composite in Back
```

Les 5 positions passées du calque sont visibles simultanément à opacité égale — effet stroboscopique.

**Usage typique — traîne colorée :**

```
Echo Operator = Add
Decay         = 0.8
```

En mode Add, les échos s'additionnent et créent des zones lumineuses saturées là où le calque est passé — idéal sur un élément lumineux (particule, néon) sur fond noir.

> 💡 Echo est un effet de **composition**, pas de rendu — il opère sur les frames telles qu'elles sont dans la timeline, keyframes et expressions incluses. Si le calque ne bouge pas, Echo ne produit rien de visible (les frames sont identiques). C'est entièrement piloté par l'animation du calque.

## Posterize Time : le look stop motion 

**Effect → Time → Posterize Time**

Posterize Time **bloque le calque à un framerate réduit**, indépendamment du framerate de la composition. Le calque se met à jour seulement N fois par seconde — ce qui donne un aspect saccadé délibéré.

```
Frame Rate → framerate cible du calque
             12 = look animation traditionnelle
             8  = look stop motion artisanal
             2  = effet très marqué, presque diaporama
```

**La différence avec `posterizeTime()` en expression :**

`posterizeTime()` en expression affecte une propriété spécifique. L'effet Posterize Time affecte **le calque entier** — son contenu visuel, ses effets, tout. Et contrairement à l'expression, l'effet est empilable avec d'autres effets dans la pile.

**Usage typique — styliser un footage en look animation 12fps :**

```
Calque footage
└── Posterize Time (Frame Rate = 12)
```

La vidéo fluide devient saccadée au rythme de l'animation traditionnelle. Souvent couplé avec un traitement colorimétrique pour renforcer l'aspect graphique.

**Usage typique — particules stop motion :**

CC Particle World génère des particules fluides. En ajoutant Posterize Time après :

```
Calque solide
├── CC Particle World
└── Posterize Time (Frame Rate = 8)
```

Les particules se déplacent par saccades — look animation d'auteur.

**Attention à l'ordre des effets :** Posterize Time doit être **après** les effets dont on veut quantifier le résultat dans la pile d'effets. Placé en premier, il quantifie le calque source mais pas les effets appliqués ensuite.

> 💡 Posterize Time est un outil de style, mais aussi un outil de performance : sur un calque génératif très lourd (Fractal Noise complexe, CC Particle World avec beaucoup de particules), abaisser à 12 ou 15fps réduit significativement le temps de calcul sans que l'œil détecte la différence dans la plupart des contextes.

## Time Difference : révéler le mouvement pur 

**Effect → Time → Time Difference**

Time Difference compare **la frame courante avec une frame décalée dans le temps** et n'affiche que leur différence — ce qui a changé. Les zones statiques deviennent noires ; seul le mouvement est visible.

```
Target Layer    → le calque à comparer avec le calque courant
                  (None = compare le calque avec lui-même à un instant différent)
Time Offset     → décalage temporel en secondes
                  valeur faible (0.03–0.1) = détecte les petits mouvements
Softness        → adoucit les bords des différences détectées
If Layer Sizes Differ → étirer ou centrer le calque de référence
Contrast        → amplifie le contraste de la différence
                  100% = résultat en niveaux de gris standard
Absolute Results → affiche la valeur absolue de la différence
                   (sans ça, des valeurs négatives peuvent apparaître en couleur)
```

**Usage typique — visualiser le mouvement dans un footage :**

```
Calque footage
└── Time Difference
    Target Layer = (None — compare avec lui-même)
    Time Offset  = 0.04   (une frame à 25fps)
    Contrast     = 200
```

Tout ce qui est statique dans le footage est noir. Les objets en mouvement apparaissent en blanc/gris selon leur vitesse.

**Usage typique — matte de mouvement :**

Le résultat de Time Difference peut être utilisé comme **Luma Matte** pour un autre calque. Cela révèle un calque seulement là où il y a du mouvement dans le footage source — une technique de compositing avancée.

```
Timeline :
├── Footage [Time Difference]      ← calque matte
└── Calque de couleur              ← TrkMat = Luma Matte
→ La couleur n'apparaît que là où le footage est en mouvement
```

**Usage typique — effet glitch/datamosh :**

Avec une valeur de Time Offset plus grande (-0.5 à -2 secondes) et le mode Echo Operator en Add, Time Difference produit des artefacts visuels qui évoquent la compression vidéo corrompue (datamosh).

<span id="widetime"></span>
## CC Wide Time : le motion blur temporel 

**Effect → Time → CC Wide Time**

CC Wide Time mélange **plusieurs frames adjacentes** — passées et futures — pour créer un motion blur purement temporel. Contrairement au Motion Blur natif d'AE qui calcule le flou à partir du mouvement du calque, CC Wide Time produit son résultat en superposant des frames réelles.

```
Forward Steps  → nombre de frames futures incluses dans le mélange
Backward Steps → nombre de frames passées incluses dans le mélange
Softness       → pondération de l'influence des frames éloignées
                 100% = toutes les frames ont le même poids
                 valeur faible = les frames les plus proches ont plus d'influence
```

**Différence avec Echo :**

- **Echo** → superpose les frames avec des opérateurs et une décroissance, produit une traîne visible
- **CC Wide Time** → mélange les frames en fondu pur, produit un flou lisse et symétrique

**Usage typique — motion blur sur un footage sans motion blur natif :**

```
Forward Steps  = 3
Backward Steps = 3
Softness       = 80
```

Le résultat est un flou de mouvement lisse centré sur la frame courante — 3 frames dans le passé et 3 frames dans le futur se fondent dans la frame actuelle.

**Usage typique — ralenti très flou (effet "rêve") :**

Sur un footage ralenti par Time Remapping :

```
Forward Steps  = 8
Backward Steps = 8
Softness       = 60
```

Les transitions entre les positions sont extrêmement fondues — effet de souvenir, de rêve, de vision subjectile.

> 💡 CC Wide Time est coûteux en performance car AE doit lire et mélanger plusieurs frames simultanément. Sur de longues compositions, le pre-render est fortement recommandé.

## Roughen Edges : bords organiques et usés 

**Effect → Stylize → Roughen Edges**

Roughen Edges déforme les bords d'un calque (ou d'un masque) de manière organique, en leur donnant un aspect rugueux, usé, brûlé ou naturel.

**Les paramètres :**

```
Edge Type    → type de déformation des bords
               Roughen / Roughen Color / Cut / Spiky / Rusty /
               Photocopy / Brush Strokes
Edge Color   → couleur ajoutée sur certains types (Roughen Color, Rusty)
Border       → distance depuis le bord où la déformation s'applique
Edge Sharpness → netteté des bords déformés (0 = très doux / 10 = très net)
Fractal Influence → influence du bruit fractal sur la déformation
                    valeur haute = irrégularité maximale
Scale        → taille des irrégularités
Stretch W/H  → étire le bruit horizontalement ou verticalement
Offset       → décale la position du bruit (pour l'animer)
Complexity   → nombre de couches de bruit (1–10)
Evolution    → fait évoluer le bruit dans le temps
               expression : time * 40
```

**Evolution — la clé pour des bords animés :**

```javascript
// Sur le paramètre Evolution :
time * 30
// Bords qui bougent lentement et organiquement
// Augmenter le multiplicateur = bords plus nerveux
```

**Usage typique — texte brûlé qui se consume :**

```
Calque texte blanc sur fond noir
├── Roughen Edges
│   Edge Type   = Rusty
│   Border      = 15
│   Evolution   → time * 50
└── [Mode Screen sur la composition]
```

**Usage typique — vieille pellicule, archives dégradées :**

```
Calque d'ajustement sur toute la comp
└── Roughen Edges
    Edge Type         = Roughen
    Border            = 8
    Fractal Influence = 0.8
    Evolution         → time * 20
```

Les bords de la composition entière tremblent légèrement — effet de vieux film projeté sur une pellicule abîmée.

**Usage typique — bords de pinceau sur un masque :**

```
Shape Layer (ellipse masque)
└── Roughen Edges
    Edge Type   = Brush Strokes
    Border      = 20
    Evolution   → time * 15
```

Le contour lisse du masque prend l'apparence de coups de pinceau irréguliers.

> 💡 Roughen Edges s'applique aux **bords du calque entier** (sa bounding box), pas à un masque spécifique. Pour cibler uniquement un masque, il faut précomposer le calque masqué avant d'appliquer Roughen Edges.

<span id="scribble"></span>
## Scribble : le remplissage gribouillis 

**Effect → Generate → Scribble**

Scribble remplit une zone de masque avec des **traits qui simulent un gribouillis animé** — comme si quelqu'un colorait rapidement à l'intérieur d'une forme.

**Les paramètres :**

```
Mask            → quel masque du calque est rempli
Fill Type       → Inside / Outside / On Edge / Across Edge
Color           → couleur des traits
Angle           → direction des traits (0° = horizontal, 90° = vertical)
Stroke Width    → épaisseur de chaque trait
Curviness       → courbure des traits
                  0 = traits droits / valeur haute = traits très courbes
Curviness Variation → aléatoire sur la courbure

Spacing         → espacement entre les traits
Spacing Variation → aléatoire sur l'espacement

Left Margin / Right Margin → marges du remplissage dans le masque

Start Frequency / End Frequency → fréquence d'oscillation des traits
```

**Wiggle** (section en bas de l'effet) :

```
Wiggles/Second  → fréquence d'animation du gribouillis
Wiggle Amount   → amplitude de l'animation
                  0 = gribouillis statique
                  valeur haute = traits qui bougent beaucoup
Correlation     → cohérence du mouvement (1.0 = tout bouge ensemble)
Temporal Phase  → décalage temporel de l'animation
Random Seed     → graine aléatoire — change l'apparence sans changer les paramètres
```

**Usage typique — révélation de titre style tableau blanc :**

```
Calque texte avec masque sur chaque lettre (ou texte converti en masques)
└── Scribble
    Fill Type    = Inside
    Color        = blanc (sur fond noir) ou couleur de la charte
    Angle        = 45°
    Stroke Width = 2
    Spacing      = 8
    Wiggles/Sec  = 2
    Wiggle Amount = 5
```

Animer **End Frequency** ou les **Margins** pour révéler progressivement le remplissage lettre par lettre.

**Usage typique — hachures d'illustration :**

```
Fill Type = Inside
Angle     = -45°
Spacing   = 12
Stroke Width = 1.5
Wiggles/Sec  = 0   → statique, pas d'animation
→ Rendu typique des hachures d'illustration technique ou de BD
```

> 💡 Scribble nécessite un **masque sur le calque** pour fonctionner. Sans masque, l'effet ne produit rien. Si on veut appliquer Scribble à la forme d'un texte, il faut d'abord convertir le texte en masques : calque texte sélectionné → `Layer → Create Masks from Text`.

<span id="repetile"></span>
## CC Repetile : répéter et étendre un calque 

**Effect → Distort → CC Repetile**

CC Repetile répète les pixels d'un calque pour **l'étendre au-delà de ses bords**, en tiles (carreaux). C'est l'outil natif pour créer des textures répétées ou étendre un calque trop petit pour remplir la composition.

**Les paramètres :**

```
Expand Right / Left / Up / Down → nombre de pixels d'extension dans chaque direction
                                   (indépendants — on peut étendre asymétriquement)
Tiling                          → mode de répétition
  Unfold   → répétition en miroir — chaque tile est le reflet du précédent
              (élimine les coutures visibles aux bords)
  Tile     → répétition directe — les tiles se raccordent bord à bord
  Reflect  → similaire à Unfold avec une légère différence de calcul
  Puzzle   → assemblage en quinconce (décalage d'une demi-tuile en Y)
```

**Usage typique — étendre une texture trop petite :**

```
Calque texture bois (800×600) dans une comp 1920×1080
└── CC Repetile
    Expand Right = 600
    Expand Left  = 520
    Expand Up    = 240
    Expand Down  = 240
    Tiling       = Unfold
→ La texture remplit toute la composition sans couture visible
```

**Usage typique — motif répété infini :**

```
Shape Layer (petit motif géométrique, 200×200px)
└── CC Repetile
    Expand Right/Left/Up/Down = 2000 (valeur large)
    Tiling = Tile
→ Motif qui remplit toute la composition comme un papier peint
```

**Animation — motif qui défile :**

```javascript
// Sur le calque avec CC Repetile, expression sur Position :
[value[0] + time * 50, value[1]]
// Le calque se déplace, CC Repetile remplit les trous créés par le déplacement
// → motif qui défile à l'infini horizontalement
```

> 💡 CC Repetile est le plus simple pour les tiles basiques. Pour des patterns complexes avec des symétries multiples (rotations, 6 axes…), le plugin **Tiler** de Motion Boutique ou une approche avec des expressions et des précomps sera plus adapté.

<span id="mosaic"></span>
## Mosaic : la pixelisation contrôlée 

**Effect → Stylize → Mosaic**

Mosaic pixelise un calque en **blocs de couleur uniforme**, chaque bloc ayant la couleur moyenne des pixels qu'il couvre. C'est la pixelisation native d'AE, bien plus contrôlée que son usage habituel (flouter des visages).

**Les paramètres :**

```
Horizontal Blocks → nombre de blocs sur la largeur
Vertical Blocks   → nombre de blocs sur la hauteur
Sharp Colors      → active/désactive l'antialiasing entre les blocs
                    activé = blocs nets avec bords durs
                    désactivé = blocs légèrement adoucis aux bords
```

**La relation entre blocs et taille :**

Plus le nombre de blocs est grand, plus les blocs sont petits — l'image est finement pixelisée. Moins il y a de blocs, plus les blocs sont larges.

```
Horizontal Blocks = 5, Vertical Blocks = 3
→ Très peu de blocs, très grands → image réduite à quelques rectangles de couleur

Horizontal Blocks = 960, Vertical Blocks = 540
→ Blocs de 2px (comp 1920×1080) → pixelisation très fine, presque invisible
```

**Usage typique — révélation en pixels :**

Animer Horizontal Blocks et Vertical Blocks de très faible (5–10) à la résolution complète (960/540 pour une comp 1080p) :

```javascript
// Expression sur Horizontal Blocks :
var t = ease(time, 0, 3, 5, 960);
Math.round(t)
// → progression de 5 blocs larges à 960 blocs fins en 3 secondes
// → l'image "se résout" progressivement depuis des pixels géants
```

**Usage typique — stylisation graphique :**

```
Horizontal Blocks = 80
Vertical Blocks   = 45
Sharp Colors      = ✅
```

L'image ressemble à un pixel art — un nombre de blocs qui correspond au format d'un écran vintage (type Game Boy, écran 8 bits).

**Usage typique — mosaïque animée comme transition :**

```
Frame 0   : Horizontal/Vertical Blocks = 960/540 (image nette)
Frame 15  : Horizontal/Vertical Blocks = 10/6 (image en gros blocs)
→ couper → nouvelle scène
Frame 0   : Horizontal/Vertical Blocks = 10/6
Frame 15  : Horizontal/Vertical Blocks = 960/540 (image résolue)
```

> 💡 Mosaic est aussi utile comme **source pour d'autres effets**. Un Mosaic sur un calque de couleur suivi d'un Turbulent Displace produit des motifs géométriques animés très différents d'un simple bruit. Mosaic sur un Fractal Noise produit une pixelisation du bruit — une texture de blocs de couleur qui évolue dans le temps.
