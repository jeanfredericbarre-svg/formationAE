## La logique conditionnelle dans AE 

Jusqu'ici, les expressions qu'on a vues produisent toujours le même type de calcul — une valeur qui évolue en continu. La logique conditionnelle permet quelque chose de différent : **changer de comportement selon une condition**. Si telle chose est vraie, faire ceci. Sinon, faire cela.

C'est ce qui permet, sans aucun keyframe :
- qu'un calque apparaisse uniquement quand un slider dépasse un seuil
- qu'une couleur bascule entre deux valeurs à un instant précis
- qu'une animation se comporte différemment selon la position d'un autre calque

Dans les expressions AE, deux syntaxes permettent d'écrire des conditions : le **if/else** et l'**opérateur ternaire**. Les deux font la même chose — le choix entre les deux est une question de lisibilité selon la complexité de la condition.

---
<span id="ifelse"></span>
## Le if/else 

C'est la syntaxe la plus lisible pour les conditions complexes ou qui impliquent plusieurs lignes de code.

**Structure de base :**

```javascript
if (condition) {
    // ce qu'on fait si la condition est vraie
} else {
    // ce qu'on fait si la condition est fausse
}
```

**Exemple concret — changer la valeur selon un slider :**

```javascript
var seuil = effect("Seuil")("Curseur");

if (seuil > 50) {
    100    // opacité à 100 si le slider dépasse 50
} else {
    0      // opacité à 0 sinon
}
```

**Exemple avec plusieurs cas — if / else if / else :**

```javascript
var etat = effect("Etat")("Curseur");

if (etat < 33) {
    [1, 0, 0, 1]       // rouge
} else if (etat < 66) {
    [1, 1, 0, 1]       // jaune
} else {
    [0, 1, 0, 1]       // vert
}
```

> 💡 Dans une expression AE, la **dernière valeur évaluée** est celle qui est retournée à la propriété. Il n'y a pas besoin d'écrire `return` — AE prend simplement la dernière expression calculée dans le bloc exécuté.

<span id="ternaire"></span>
## L'opérateur ternaire 

L'opérateur ternaire est une façon compacte d'écrire un if/else sur une seule ligne. Il est très répandu dans les expressions AE car la plupart des conditions sont simples et tiennent facilement en une ligne.

**Syntaxe :**

```javascript
condition ? valeurSiVraie : valeurSiFausse
```

**Exemple concret — la même condition que ci-dessus, en une ligne :**

```javascript
var seuil = effect("Seuil")("Curseur");
seuil > 50 ? 100 : 0
```

**Exemple — afficher ou masquer selon le temps :**

```javascript
time > 2 ? 100 : 0    // opacité passe à 100 après 2 secondes
```

**Exemple — inverser une valeur selon un interrupteur :**

```javascript
var inverse = effect("Inverser")("Case à cocher");
inverse ? -value[0] : value[0]    // inverse le X si la case est cochée
```

Les opérateurs ternaires peuvent être imbriqués, mais au-delà de deux niveaux la lisibilité chute rapidement — dans ce cas, préfère un `if / else if / else` :

```javascript
// Lisible à deux niveaux
etat < 50 ? 0 : etat < 75 ? 50 : 100

// Difficile à lire à trois niveaux — préfère if/else ici
etat < 25 ? 0 : etat < 50 ? 33 : etat < 75 ? 66 : 100
```

> 💡 Choix entre les deux syntaxes : utilise l'**opérateur ternaire** pour les conditions simples à une valeur de retour. Utilise **if/else** dès que tu as plusieurs lignes de calcul dans chaque branche, ou plus de deux cas.

<span id="comparaison"></span>
## Les opérateurs de comparaison 

Une condition est toujours une expression qui retourne `true` (vrai) ou `false` (faux). Les opérateurs de comparaison permettent de construire ces conditions.

| Opérateur | Signification | Exemple | Résultat |
|---|---|---|---|
| `===` | égal à | `etat === 1` | vrai si etat vaut exactement 1 |
| `!==` | différent de | `etat !== 0` | vrai si etat est différent de 0 |
| `>` | supérieur à | `time > 2` | vrai après 2 secondes |
| `<` | inférieur à | `time < 5` | vrai avant 5 secondes |
| `>=` | supérieur ou égal | `index >= 3` | vrai à partir du 3ème calque |
| `<=` | inférieur ou égal | `value[1] <= 0` | vrai si Y est à zéro ou négatif |

**L'erreur classique — confondre `=` et `===` :**

```javascript
// FAUX — ceci est une assignation, pas une comparaison
if (etat = 1) { ... }

// CORRECT — ceci est une comparaison
if (etat === 1) { ... }
```

Utiliser `=` dans une condition ne génère pas toujours une erreur visible dans AE — la condition sera évaluée à `true` dans la plupart des cas, ce qui crée un bug silencieux difficile à repérer.

> 💡 En JavaScript moderne on distingue `==` (égalité laxiste, avec conversion de type) et `===` (égalité stricte). Dans les expressions AE, utilise toujours `===` et `!==` — c'est plus prévisible et c'est la bonne habitude à prendre.

<span id="combiner"></span>
## Combiner des conditions 

On peut combiner plusieurs conditions avec les opérateurs logiques :

| Opérateur | Signification | Exemple |
|---|---|---|
| `&&` | ET — les deux doivent être vraies | `time > 1 && time < 3` |
| `\|\|` | OU — au moins une doit être vraie | `etat === 1 \|\| etat === 3` |
| `!` | NON — inverse la condition | `!actif` (vrai si actif est faux) |

**Exemple — une valeur active uniquement dans une fenêtre de temps :**

```javascript
time > 1 && time < 4 ? 100 : 0
// opacité à 100 uniquement entre 1s et 4s
```

**Exemple — réagir à plusieurs états possibles :**

```javascript
var etat = effect("Etat")("Curseur");
etat === 1 || etat === 3 ? 100 : 0
// opacité à 100 si etat vaut 1 ou 3
```

**Exemple — condition négative :**

```javascript
var actif = effect("Actif")("Case à cocher");
!actif ? 100 : 0
// opacité à 100 quand la case est décochée (inverseur)
```

> 💡 Les conditions combinées se lisent comme une phrase : `time > 1 && time < 4` se lit "le temps est supérieur à 1 ET inférieur à 4". Si la phrase est claire à voix haute, la condition est bien écrite.

<span id="time"></span>
## Combiner avec time 

`time` est la variable qui contient le temps courant en secondes. Combinée à des conditions, elle permet de créer des comportements qui changent à des instants précis — sans aucun keyframe.

**Déclencher un changement à un instant précis :**

```javascript
time > 2 ? 100 : 0
// bascule instantanément de 0 à 100 à exactement 2 secondes
```

**Séquencer plusieurs états :**

```javascript
if (time < 1) {
    0
} else if (time < 3) {
    100
} else if (time < 5) {
    50
} else {
    0
}
```

**Combiner condition et interpolation pour une transition douce :**

```javascript
// Apparition progressive entre 1s et 2s, puis disparition entre 4s et 5s
if (time < 1) {
    0
} else if (time < 2) {
    linear(time, 1, 2, 0, 100)
} else if (time < 4) {
    100
} else if (time < 5) {
    linear(time, 4, 5, 100, 0)
} else {
    0
}
```

> 💡 Cette technique remplace avantageusement une série de keyframes d'opacité quand le timing est défini par des règles plutôt que par des positions précises dans la timeline. Elle est aussi beaucoup plus facile à modifier : changer un chiffre suffit à décaler toute la séquence.

<span id="pratiques"></span>
## Cas pratiques 

**Afficher un calque uniquement quand un slider dépasse un seuil :**

```javascript
// Sur la propriété Opacité
var seuil = effect("Seuil")("Curseur");
transform.position[1] < seuil ? 100 : 0
// visible uniquement si le Y du calque est au-dessus du seuil
```

**Interrupteur marche/arrêt via une case à cocher :**

```javascript
// Sur la propriété Opacité
var actif = effect("Visible")("Case à cocher");
actif ? 100 : 0
```

**Changer de couleur au passage d'un seuil de vitesse :**

```javascript
// Sur la propriété Couleur d'un effet Remplissage
var vitesse = Math.abs(thisComp.layer("OBJET").transform.position.velocityAtTime(time)[0]);
vitesse > 200 ? [1, 0, 0, 1] : [0, 1, 0, 1]
// rouge si l'objet va vite, vert sinon
```

**Alterner entre deux valeurs à chaque seconde :**

```javascript
// Sur n'importe quelle propriété numérique
Math.floor(time) % 2 === 0 ? 0 : 100
// 0 pendant les secondes paires, 100 pendant les secondes impaires
```

**Séquencer une animation en trois phases basées sur l'index du calque :**

```javascript
// Sur la propriété Opacité — chaque calque apparaît 0.2s après le précédent
var delai = (thisLayer.index - 1) * 0.2;
var t = time - delai;

t < 0 ? 0 : t < 0.5 ? linear(t, 0, 0.5, 0, 100) : 100
```
