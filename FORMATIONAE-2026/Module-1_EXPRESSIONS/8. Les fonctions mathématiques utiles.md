# L'objet Math 

Toutes les fonctions mathématiques de JavaScript sont regroupées dans un objet appelé `Math`. Cet objet existe nativement dans AE, pas besoin de l'importer ou de le déclarer. On y accède toujours avec la syntaxe `Math.nomDeLaFonction()`.

```javascript
Math.round(3.7)   // → 4
Math.abs(-50)     // → 50
Math.PI           // → 3.14159...
```

> 💡 Analogie : `Math` est une calculatrice scientifique toujours disponible sur ton bureau. Tu n'as pas à la brancher, elle est là, tu t'en sers quand tu en as besoin.

Ces fonctions ne sont pas spécifiques à AE : elles font partie du JavaScript standard. Elles fonctionnent donc exactement de la même manière que dans n'importe quel contexte JS.
Tu peux retrouver toutes les fonctions Math de Javascript sur cette page : https://www.w3schools.com/jsref/jsref_obj_math.asp

# Arrondir des valeurs 

Trois fonctions permettent d'arrondir un nombre décimal. Elles sont indispensables dès qu'on travaille avec des valeurs calculées qu'on ne veut pas afficher avec 12 décimales, ou qu'on veut contraindre à des nombres entiers.

## Math.round() : arrondi classique

Arrondit au nombre entier **le plus proche**. En dessous de 0.5, arrondit vers le bas ; à partir de 0.5, arrondit vers le haut.

```javascript
Math.round(3.2)   // → 3
Math.round(3.5)   // → 4
Math.round(3.8)   // → 4
Math.round(-1.5)  // → -1
```

## Math.floor() : arrondi vers le bas

Arrondit **toujours vers le bas**, vers l'entier inférieur. En français : "le plancher".

```javascript
Math.floor(3.9)   // → 3   (pas 4 !)
Math.floor(3.1)   // → 3
Math.floor(-1.2)  // → -2  (attention : vers le bas, pas vers zéro)
```

**Usage typique :** calculer un numéro de frame à partir du temps.

```javascript
Math.floor(time * thisComp.frameRate)
// time = 1.04s, frameRate = 25 → Math.floor(26.0) → 26
// Donne un index entier utilisable pour indexer un tableau ou afficher un numéro de frame
```

## Math.ceil() : arrondi vers le haut

Arrondit **toujours vers le haut**, vers l'entier supérieur. En français : "le plafond".

```javascript
Math.ceil(3.1)    // → 4   (pas 3 !)
Math.ceil(3.9)    // → 4
Math.ceil(-1.9)   // → -1  (vers le haut, donc vers zéro)
```

## Arrondir à N décimales

Ces trois fonctions n'arrondissent qu'aux entiers. Pour arrondir à 2 décimales par exemple, on combine avec une multiplication :

```javascript
// Arrondir à 2 décimales :
Math.round(value * 100) / 100
// value = 3.14159 → Math.round(314.159) → 314 → / 100 → 3.14
```

# Valeurs absolues et signes 

## Math.abs() : valeur absolue

Retourne la valeur **sans son signe**, toujours positive ou nulle.

```javascript
Math.abs(50)    // → 50
Math.abs(-50)   // → 50
Math.abs(0)     // → 0
```

**Usage typique :** mesurer une distance ou une amplitude, peu importe la direction.

```javascript
// Distance entre la position X du calque courant et le centre de la comp :
const dist = Math.abs(thisLayer.transform.position[0] - thisComp.width / 2);
// → toujours positive, que le calque soit à gauche ou à droite du centre
```

## Math.sign() : le signe d'un nombre

Retourne `1` si le nombre est positif, `-1` s'il est négatif, `0` s'il est nul. Utile pour "extraire" la direction d'une valeur.

```javascript
Math.sign(45)    // → 1
Math.sign(-45)   // → -1
Math.sign(0)     // → 0
```

**Usage typique :** appliquer un effet dans la même direction qu'une valeur, quelle qu'elle soit.

```javascript
// Rotation qui suit le signe de la vélocité de la position X :
const vel = thisLayer.transform.position.velocityAtTime(time)[0];
Math.sign(vel) * 15
// → incline de +15° quand le calque va vers la droite, -15° vers la gauche
```

# Borner une valeur : clamp() 

`clamp()` est une **fonction propre à AE** (pas du JavaScript standard). Elle contraint une valeur dans un intervalle `[min, max]` : si la valeur dépasse le maximum, elle est ramenée au maximum ; si elle passe sous le minimum, elle est ramenée au minimum.

```javascript
clamp(value, min, max)
```

```javascript
clamp(150, 0, 100)    // → 100  (valeur trop haute, ramenée au max)
clamp(-20, 0, 100)    // → 0    (valeur trop basse, ramenée au min)
clamp(60, 0, 100)     // → 60   (dans l'intervalle, valeur inchangée)
```

**Usage typique :** s'assurer qu'une valeur calculée dynamiquement ne sort jamais d'une plage acceptable.

```javascript
// Opacité qui suit la position X mais ne dépasse jamais 0-100 :
const x = thisLayer.transform.position[0];
clamp(x / 10, 0, 100)
// Si x = 500 → 50 ✅ | si x = 1500 → 100 (pas 150) ✅ | si x = -200 → 0 ✅
```

> 💡 `clamp()` est l'équivalent des garde-fous sur une route de montagne. La voiture peut aller où elle veut, mais elle ne tombera jamais dans le vide.

# Math.min() et Math.max() 

Ces deux fonctions retournent respectivement la **plus petite** ou la **plus grande** valeur parmi une liste de nombres. Elles acceptent autant d'arguments que nécessaire.

```javascript
Math.min(10, 30, 5, 20)   // → 5
Math.max(10, 30, 5, 20)   // → 30
```

**Usage typique :** limiter une valeur dans une direction seulement.

```javascript
// Empêcher une position Y de descendre sous 100px (plancher) :
Math.max(thisLayer.transform.position[1], 100)

// Empêcher une opacité de dépasser 80 (plafond) :
Math.min(time * 40, 80)
```

La différence avec `clamp()` : `Math.min` et `Math.max` ne bornent que d'un côté à la fois. `clamp(v, min, max)` est en fait l'équivalent de `Math.min(Math.max(v, min), max)`.

# Les fonctions trigonométriques 

`Math.sin()`, `Math.cos()` et `Math.tan()` sont les trois fonctions trigonométriques de base. En motion design, `sin` et `cos` sont utilisées constamment pour créer des **mouvements oscillants, circulaires ou ondulatoires**.

> 💡 Analogie : `Math.sin(time)` c'est comme regarder la hauteur d'un point sur une roue qui tourne. La roue tourne à vitesse constante, le point monte et descend régulièrement entre -1 et +1.

## Ce qu'elles prennent en entrée

Ces fonctions attendent un angle exprimé en **radians**, pas en degrés. C'est la source de confusion la plus fréquente.

```javascript
// 1 radian = environ 57.3°
// Un tour complet = 2 * Math.PI radians = 360°

Math.sin(0)           // → 0
Math.sin(Math.PI / 2) // → 1   (correspond à 90°)
Math.sin(Math.PI)     // → 0   (correspond à 180°)
Math.sin(Math.PI * 2) // → 0   (correspond à 360° = retour au départ)
```

## Ce qu'elles retournent

`Math.sin()` et `Math.cos()` retournent toujours une valeur **entre -1 et +1**. Pour obtenir une amplitude utile, il faut multiplier par l'amplitude souhaitée.

## Oscillation simple avec Math.sin()

```javascript
// Sur la propriété Position Y — oscillation verticale :
const freq = 2;        // vitesse : 2 cycles par seconde
const amp = 50;        // amplitude : 50px de part et d'autre
value + Math.sin(time * freq * Math.PI * 2) * amp
// value → position Y de base (avec ou sans keyframes)
// + sin(...) → ajoute une oscillation par-dessus
```

## Mouvement circulaire avec sin et cos ensemble

`sin` et `cos` sont déphasés de 90° l'un par rapport à l'autre. Combinés sur X et Y, ils produisent un **mouvement circulaire parfait**.

```javascript
// Sur la propriété Position — mouvement circulaire autour du centre de la comp :
const cx = thisComp.width / 2;   // centre X
const cy = thisComp.height / 2;  // centre Y
const rayon = 150;
const freq = 0.5;                 // demi-tour par seconde

[
  cx + Math.cos(time * freq * Math.PI * 2) * rayon,
  cy + Math.sin(time * freq * Math.PI * 2) * rayon
]
```

## Différence entre sin et cos

```javascript
Math.sin(0)   // → 0   (commence au milieu)
Math.cos(0)   // → 1   (commence au maximum)
```

En pratique : utiliser `sin` quand l'animation doit démarrer à sa **valeur neutre** (au milieu de l'oscillation), utiliser `cos` quand elle doit démarrer à son **maximum**.

# Math.PI et les conversions d'angles 

`Math.PI` est une constante, la valeur de π (pi), soit environ `3.14159265358979`. Elle est indispensable dès qu'on travaille avec des angles ou des fonctions trigonométriques.

## Convertir degrés → radians

Les fonctions `sin`, `cos`, `tan` attendent des radians. Pour leur passer des degrés, on convertit :

```javascript
const deg = 45;
const rad = deg * Math.PI / 180;
Math.sin(rad)   // → 0.707...
```

**Raccourci courant :**

```javascript
// Rotation de 45° exprimée en radians :
Math.sin(45 * Math.PI / 180)
```

## Convertir radians → degrés

Dans l'autre sens — par exemple pour afficher une valeur en degrés dans un sourceText :

```javascript
const rad = Math.PI / 4;            // π/4 radians
const deg = rad * 180 / Math.PI;    // → 45°
```

## Tours complets

Un tour complet vaut `2 * Math.PI` radians. C'est la valeur à utiliser comme période dans les oscillations :

```javascript
Math.sin(time * Math.PI * 2)
// → accomplit un cycle complet (de 0 à 2π) en exactement 1 seconde
```

# Math.pow() et Math.sqrt() 

## Math.pow() : puissance

`Math.pow(base, exposant)` élève un nombre à une puissance donnée.

```javascript
Math.pow(2, 3)    // → 8    (2³)
Math.pow(10, 2)   // → 100  (10²)
Math.pow(4, 0.5)  // → 2    (racine carrée, équivalent à Math.sqrt(4))
```

**Usage typique :** courbes d'accélération non linéaires. Une progression quadratique (`pow(t, 2)`) accélère plus naturellement qu'une progression linéaire.

```javascript
// Opacité qui s'ouvre selon une courbe quadratique entre 0 et 2 secondes :
const t = clamp(time / 2, 0, 1);   // normalise time entre 0 et 1
Math.pow(t, 2) * 100
// → 0% à t=0, 25% à t=0.5, 100% à t=1 — accélération progressive
```

## Math.sqrt() : racine carrée

`Math.sqrt(n)` retourne la racine carrée de `n`.

```javascript
Math.sqrt(9)    // → 3
Math.sqrt(2)    // → 1.4142...
Math.sqrt(0)    // → 0
```

**Usage typique :** calculer une distance euclidienne (théorème de Pythagore) entre deux points.

```javascript
// Distance entre deux calques dans la composition :
const p1 = thisLayer.transform.position;
const p2 = thisComp.layer("Cible").transform.position;
const dx = p2[0] - p1[0];
const dy = p2[1] - p1[1];
Math.sqrt(dx * dx + dy * dy)
// → distance en pixels entre les deux calques, mise à jour à chaque frame
```

# Math.random() 

`Math.random()` retourne un nombre aléatoire entre `0` (inclus) et `1` (exclus), différent à chaque évaluation.

```javascript
Math.random()   // → 0.472... ou 0.891... ou 0.013... — imprévisible
```

**Usage typique :** générer une valeur aléatoire dans une plage personnalisée.

```javascript
// Nombre aléatoire entre min et max :
const min = 20;
const max = 80;
Math.random() * (max - min) + min
// → valeur aléatoire entre 20 et 80
```

## Math.random() vs random() vs wiggle()

AE propose ses propres fonctions aléatoires, il est important de distinguer leur comportement :

```javascript
Math.random()   // JS standard — recalculé à chaque frame ET à chaque rendu
                // → résultat différent à chaque rendu : instable, déconseillé sur les propriétés animées

random()        // Propre à AE — recalculé à chaque frame, mais stable entre les rendus
                // → résultat cohérent d'un rendu à l'autre pour la même frame

wiggle()        // Propre à AE — interpolé dans le temps (pas de saut brutal entre frames)
                // → le choix naturel pour des mouvements organiques continus
```

> 💡 Pour tout ce qui est animé et visible, préférer `random()` ou `wiggle()` à `Math.random()`. `Math.random()` est surtout utile pour générer des valeurs **initiales fixes** dans des scripts, ou pour des usages non animés.

**Générer un offset aléatoire stable avec `random()` :**

```javascript
seedRandom(index, true);   // graine basée sur l'index du calque — unique par calque
const offset = random(0, 2); // valeur entre 0 et 2 secondes, stable entre les rendus
loopOut("cycle");          // ... utilisé pour décaler une boucle par exemple
```
