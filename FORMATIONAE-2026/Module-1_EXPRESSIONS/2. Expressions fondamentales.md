<span id="valueattime"></span>
## time et valueAtTime()

### time

`time` est une variable **disponible automatiquement** dans toute expression. Elle représente la position de la tête de lecture de la composition, exprimée en **secondes**.

```javascript
time        // à la frame 0 → 0
            // à la frame 25 (comp 25fps) → 1
            // à la frame 50 → 2
```

Combiné à une opération mathématique, `time` devient une possibilité d'animation sans keyframe :

```javascript
// Sur la propriété Rotation :
time * 90   // tourne de 90° par seconde, soit un tour complet toutes les 4 secondes
```

### valueAtTime()

`valueAtTime(t)` lit la valeur d'une propriété **à un instant précis**, quel que soit la position actuelle de la tête de lecture.

**Syntaxe :**

```javascript
thisLayer.transform.position.valueAtTime(t) // t : le temps en secondes auquel on veut lire la valeur
```

**Cas d'usage : le décalage temporel.** Imaginons un calque A dont la position est animée avec des keyframes. On veut qu'un calque B suive exactement le même chemin, mais avec 0,5 secondes de retard. Sur la propriété Position du calque B :

```javascript
thisComp.layer("Calque A").transform.position.valueAtTime(time - 0.5)
// "quelle était la position de Calque A il y a 0,5 secondes ?"
```

> 💡 Cette technique permet de créer des effets de traînée ou d'écho, et des éléments qui suivent un leader avec du retard, sans dupliquer une seule keyframe.

<span id="random"></span>
## random() et gaussRandom()

Ces deux fonctions génèrent des **valeurs aléatoires ponctuelles**, c'est-à-dire une valeur différente à chaque frame. Contrairement à `wiggle()`, elles ne produisent pas un mouvement continu et interpolé : la valeur saute brusquement d'une frame à l'autre.

### random()

Sans argument, `random()` retourne un nombre entre 0 et 1. Avec des arguments, on définit la plage de sortie.

```javascript
random()            // nombre aléatoire entre 0 et 1
random(100)         // nombre aléatoire entre 0 et 100
random(50, 200)     // nombre aléatoire entre 50 et 200
random([0,0], [100,100])  // Array aléatoire, utile pour la Position
```

### gaussRandom()

Même syntaxe que `random()`, mais la distribution est **gaussienne** : les valeurs proches du centre de la plage sont beaucoup plus fréquentes que les extrêmes. Le résultat est plus naturel, moins "mécanique".

```javascript
gaussRandom(0, 100)  // le résultat sera souvent proche de 50, rarement proche de 0 ou 100
```

> 💡 Analogie : `random()` c'est un dé équitable, chaque face a autant de chances. `gaussRandom()` c'est la taille des gens dans une population : la moyenne est courante, les extrêmes sont plus rares.

### seedRandom() : figer le hasard

Par défaut, `random()` et `wiggle()` génèrent un résultat **différent à chaque rendu**. Le terme **rendu** est très important, si tu fermes ton projet et le réouvre le résultat ne sera pas le même, idem si le même projet est ouvert sur une autre machine. Ce n'est généralement pas très gênant et même à peine visible (sur un tremblement de texte par exemple) mais ça peut poser un problème si tu dois refaire un rendu d'une partie d'une composition pour l'ajouter au rendu précédent (genre modif' client).

En fixant la graine avec `seedRandom(n, true)`, le résultat sera identique quelle que soit la machine ou le moment.

```javascript
seedRandom(42, true);
random(0, 100)
// 42 est une graine arbitraire, sa valeur n'a pas d'importance,
// seule sa constance compte. true = résultat stable entre les rendus.
```
on verra ça en pratique dans le prochain projet AE à télécharger.

## wiggle()

`wiggle()` ajoute un **mouvement organique et aléatoire** à une propriété, sans aucun keyframe. Contrairement à `random()`, les valeurs sont **interpolées** : le mouvement est fluide et continu.

<img src="FORMATIONAE-2026/images/PushTheTempo.gif" alt="Fat Boy Slim - Push The Tempo" width="400">

> 🎁  Fat Boy Slim "*Push The Tempo*" 
<a href="https://www.youtube.com/watch?v=JEq10L7u3SM" target="_blank">... regarde, ça déchire !</a>

**Syntaxe complète :**

```javascript
wiggle(freq, amp, octaves, amp_mult, t)
```
Les 2 premiers paramètres sont obligatoires, les suivants sont optionnels, un wiggle minimum et que tu utiliseras la plupart du temps, c'est :
```javascript
wiggle(4, 50)
```

### Les paramètres dans l'ordre

```javascript
wiggle(freq, amp)
// freq     : obligatoire. Nombre d'oscillations par seconde.
// amp      : obligatoire. Amplitude maximale du déplacement,
//            appliquée autour de la valeur courante, pas à partir de 0.
// octaves  : optionnel, défaut = 1. Couches de bruit superposées.
//            Plus la valeur est haute, plus le mouvement est détaillé.
// amp_mult : optionnel, défaut = 0.5. Vitesse à laquelle l'amplitude diminue
//            à chaque octave supplémentaire.
// t        : optionnel, défaut = time. Instant de référence pour le calcul.
```

**Exemples concrets :**

```javascript
wiggle(5, 30)         // 5 oscillations/sec, amplitude de 30px, mouvement nerveux
wiggle(1, 50)         // 1 oscillation/sec, amplitude de 50px, mouvement lent et ample
wiggle(3, 20, 3)      // 3 octaves, mouvement plus organique et irrégulier
```

### Wiggle sur un seul axe, un grand classique !

Par défaut, `wiggle()` appliqué à la Position bouge sur X **et** Y simultanément. Pour restreindre le mouvement à un seul axe :

```javascript
const w = wiggle(3, 40);
// w est un Array [X_wiggly, Y_wiggly]

[value[0], w[1]]
// garde la valeur X d'origine (pas de wiggle sur X)
// remplace la valeur Y par la valeur Y du wiggle
```

<span id="posterizetime"></span>
## posterizeTime()

`posterizeTime(fps)` **réduit la fréquence d'évaluation** d'une expression. Là où After Effects évalue normalement l'expression à chaque frame, `posterizeTime()` la force à ne se mettre à jour qu'à la fréquence indiquée. Le résultat : un mouvement saccadé, comme une animation frame by frame.

```javascript
posterizeTime(6);
wiggle(3, 40)
// le wiggle ne se met à jour que 6 fois par seconde,
// quelle que soit la fréquence de la composition
```

> 💡 Très utile pour simuler l'esthétique d'une animation traditionnelle, ou donner un aspect "glitchy" à un texte ou à une valeur numérique.

**Attention :** `posterizeTime()` s'applique à **tout ce qui suit** dans l'expression. Il faut l'appeler en début d'expression si on veut qu'elle affecte tout le calcul.

<span id="loopoutloopin"></span>
## loopOut() et loopIn()

`loopOut()` et `loopIn()` font une seule chose : **prolonger une animation existante au-delà de ses keyframes**, sans en ajouter de nouveaux. AE répète la séquence à ta place.

La règle d'or : **il faut au minimum 2 keyframes** sur la propriété pour que la boucle ait quelque chose à répéter.

**Syntaxe complète :**

```javascript
loopOut(type, numKeyframes)
// Les deux paramètres sont optionnels.
// Par défaut : loopOut() est identique à loopOut("cycle", 0)
```

### Les 4 types

```javascript
loopOut("cycle")     // Répète la séquence du premier au dernier keyframe
loopOut("pingpong")  // Fait l'aller puis le retour, en miroir
loopOut("offset")    // Répète en décalant cumulativement la valeur
loopOut("continue")  // Ne répète pas, continue à la même vitesse indéfiniment
```

**cycle**, le plus courant. La séquence repart du début à chaque fois. Pour une boucle fluide, le dernier keyframe doit avoir la même valeur que le premier, sinon il y aura un saut visuel.

**pingpong**, la séquence joue en avant, puis en arrière, alternativement. Idéal pour un mouvement de balancier ou un effet de respiration.

**offset**, à chaque répétition, la valeur repart du point d'arrivée du cycle précédent. Parfait pour une roue qui tourne indéfiniment, ou un objet qui monte sans fin.

**continue**, ne répète rien. AE calcule la vélocité au dernier keyframe et continue dans cette direction. Utile pour simuler un objet qui sort du cadre naturellement.

### Le paramètre numKeyframes

`numKeyframes` ne désigne pas un nombre de keyframes mais un nombre de **segments**.

```javascript
loopOut("cycle", 0)  // boucle sur TOUS les keyframes (défaut)
loopOut("cycle", 1)  // boucle sur le dernier segment, 2 keyframes inclus
loopOut("cycle", 2)  // boucle sur les 2 derniers segments, 3 keyframes inclus
```

> 💡 Règle simple : `numKeyframes = N` → `N+1` keyframes sont inclus dans la boucle.

### loopOutDuration()

Variante où la boucle est définie par une **durée en secondes** plutôt que par un nombre de segments.

```javascript
loopOutDuration("cycle", 2)  // boucle sur les 2 dernières secondes d'animation
```

<span id="linear"></span>
## linear(), ease(), easeIn(), easeOut()

Ces quatre fonctions font **une seule et unique chose** : prendre une valeur d'entrée dans une plage connue, et la **convertir proportionnellement** vers une plage de sortie. Ce qui les distingue, c'est la **courbe** de cette conversion.

> 💡 Analogie : c'est exactement ce que fait une règle de trois, mais avec la possibilité d'adoucir la transition.

### La syntaxe, argument par argument

```javascript
linear(t, tMin, tMax, value1, value2)
// t      : la valeur d'entrée (souvent time, ou value, ou n'importe quel nombre)
// tMin   : la valeur minimale de t (début de la plage d'entrée)
// tMax   : la valeur maximale de t (fin de la plage d'entrée)
// value1 : la valeur de sortie quand t = tMin
// value2 : la valeur de sortie quand t = tMax
```

**Exemple concret :** piloter l'opacité d'un calque entre les secondes 1 et 3.

```javascript
linear(time, 1, 3, 0, 100)
// quand time = 1  → opacité = 0
// quand time = 2  → opacité = 50 (progression linéaire)
// quand time = 3  → opacité = 100
// avant 1 et après 3 : AE clamp automatiquement (0 avant, 100 après)
```

### Les 4 variantes

```javascript
linear(t, tMin, tMax, v1, v2)   // progression mécanique, vitesse constante
ease(t, tMin, tMax, v1, v2)     // ralentit au début ET à la fin (easeInOut)
easeIn(t, tMin, tMax, v1, v2)   // ralentit uniquement au début
easeOut(t, tMin, tMax, v1, v2)  // ralentit uniquement à la fin
```

**Raccourci :** si la plage d'entrée est `[0, 1]`, la syntaxe courte existe :

```javascript
linear(t, v1, v2)   // équivalent à linear(t, 0, 1, v1, v2)
```

### t n'est pas forcément time

`t` peut être **n'importe quelle valeur numérique**, pas seulement le temps.

```javascript
// Piloter la rotation depuis la valeur d'un Slider Control :
const slider = effect("Mon Slider")("Curseur");
linear(slider, 0, 100, -45, 45)
// quand le slider vaut 0   → rotation = -45°
// quand le slider vaut 100 → rotation = +45°
```

<span id="clamp"></span>
## clamp()

`clamp()` **bloque une valeur dans une plage** définie par un minimum et un maximum. Si la valeur dépasse le maximum, elle est ramenée au maximum. Si elle descend sous le minimum, elle est ramenée au minimum.

```javascript
clamp(value, min, max)
// value : la valeur à contraindre
// min   : la borne inférieure
// max   : la borne supérieure
```

**Exemples :**

```javascript
clamp(150, 0, 100)   // → 100  (150 dépasse le max)
clamp(-20, 0, 100)   // → 0    (-20 est sous le min)
clamp(50, 0, 100)    // → 50   (50 est dans la plage, pas de changement)
```

**Cas d'usage concret :** empêcher un calque de sortir de la composition.

```javascript
// Sur la propriété Position X du calque :
const x = thisComp.layer("CONTROLS").transform.position[0];
clamp(x, 0, thisComp.width)
// la position X ne peut jamais dépasser les bords gauche et droit de la comp
```

> 💡 `clamp()` est souvent utilisé en complément de `linear()` pour s'assurer que le résultat reste dans des bornes raisonnables, même si la valeur d'entrée dépasse la plage prévue.

<span id="length"></span>
## length()

`length()` calcule la **distance euclidienne** (en ligne droite) entre deux points. Elle accepte des Arrays 2D ou 3D.

```javascript
length(point1, point2)
// retourne la distance en pixels entre les deux points

length(value)
// forme courte : distance entre [0,0] et la valeur courante (= magnitude du vecteur)
```

**Cas d'usage concret :** faire varier l'opacité d'un calque en fonction de sa distance à un autre calque.

```javascript
const posA = thisLayer.transform.position;
const posB = thisComp.layer("Cible").transform.position;
const dist = length(posA, posB);

// l'opacité diminue quand le calque s'éloigne de "Cible"
linear(dist, 0, 500, 100, 0)
```

> 💡 `length()` est la fonction de base pour tout ce qui implique une notion de proximité, d'attraction ou de répulsion entre calques.

<span id="degres"></span>
## degreesToRadians() et radiansToDegrees() ... en avant les maths !

Les fonctions trigonométriques JavaScript (`Math.sin()`, `Math.cos()`) travaillent en **radians**, mais After Effects raisonne en **degrés**. Ces deux fonctions font la conversion.

```javascript
degreesToRadians(deg)   // convertit des degrés en radians
radiansToDegrees(rad)   // convertit des radians en degrés
```

**Rappel de conversion :**

```
360° = 2π radians
180° = π radians
90°  = π/2 radians
```

> 💡 En pratique, on utilise presque toujours `degreesToRadians()` avant de passer une valeur à `Math.sin()` ou `Math.cos()`, et `radiansToDegrees()` quand on veut afficher ou utiliser un résultat trigonométrique comme un angle dans AE.

<span id="sincos"></span>
## Math.sin() et Math.cos()

`Math.sin()` et `Math.cos()` produisent des **oscillations parfaitement régulières et contrôlées**, entre -1 et 1. C'est l'alternative mathématique au `wiggle()` : là où `wiggle()` est organique et imprévisible, `sin` et `cos` sont précis et déterministes.

> 💡 Analogie : `wiggle()` c'est une mer agitée, `Math.sin()` c'est une vague parfaitement sinusoïdale en piscine.

### La structure de base

```javascript
Math.sin(time * vitesse) * amplitude
// time * vitesse : contrôle la fréquence de l'oscillation
// * amplitude    : contrôle l'intensité du déplacement
```

**Exemple sur la Position Y :**

```javascript
const freq = 2;        // on définit 2 cycles par seconde
const amp  = 50;       // on définit 50 pixels d'amplitude
[value[0], value[1] + Math.sin(time * freq * Math.PI * 2) * amp]
// le calque monte et descend régulièrement de 50px
```

La ligne qui fait tout le travail, décortiquée étape par étape :

```javascript
Math.sin(time * freq * Math.PI * 2) * amp
```

---

**`time`** — le temps qui s'écoule, en secondes. C'est la variable qui fait avancer le mouvement. À chaque frame, `time` augmente, ce qui fait progresser le calcul.

---

**`freq`** — le nombre de cycles par seconde (ici `2`). Multiplier `time` par `freq` accélère ou ralentit l'oscillation :

- `freq = 1` → un aller-retour complet en 1 seconde
- `freq = 2` → deux allers-retours en 1 seconde
- `freq = 0.5` → un aller-retour toutes les 2 secondes

---

**`Math.PI * 2`** — c'est la conversion en **radians**. `Math.sin()` ne raisonne pas en degrés, mais en radians, et un cercle complet vaut `2π` radians. Sans ce facteur, `Math.sin(time)` ferait bien une oscillation, mais _très_ lente (un cycle en ~6,28 secondes, soit la valeur de 2π).
En combinant les trois : `time * freq * Math.PI * 2` donne à `Math.sin()` une valeur en radians qui progresse exactement au rythme voulu.

---

**`Math.sin(...)`** — la fonction sinus. Elle reçoit cette valeur en radians et retourne toujours un nombre **entre −1 et +1**, en décrivant une courbe en S parfaitement régulière. C'est ce qui crée le mouvement fluide, sans à-coups.

---

**`* amp`** — l'amplitude (ici `50`). Le résultat de `Math.sin()` est minuscule (entre −1 et +1), il faut l'amplifier pour que ça ait un effet visible. Multiplier par `50` étire cette plage à **−50 / +50 pixels**.

---

Pour visualiser l'ensemble :

|Moment|`time`|`...* freq * Math.PI * 2`|`Math.sin(...)`|`* amp`|
|---|---|---|---|---|
|Départ|0 s|0|0|**0 px**|
|Quart de cycle|0.25 s|π/2|1|**+50 px**|
|Demi-cycle|0.5 s|π|0|**0 px**|
|Trois quarts|0.75 s|3π/2|−1|**−50 px**|
|Cycle complet|1 s|2π|0|**0 px**|


### sin vs cos : le déphasage

`Math.cos()` est identique à `Math.sin()` mais **décalé d'un quart de cycle**. En combinant les deux sur X et Y, on obtient un mouvement circulaire :

```javascript
const freq = 1;
const rayon = 100;
[
  value[0] + Math.cos(time * freq * Math.PI * 2) * rayon,
  value[1] + Math.sin(time * freq * Math.PI * 2) * rayon
]
// le calque décrit un cercle parfait autour de sa position d'origine
```

> 💡 `sin` et `cos` sont les fondements de toute animation procédurale avancée : spirales, orbites, vagues en cascade, méchanismes complexes.

<span id="index"></span>
## index

`index` est une propriété disponible automatiquement sur tout calque. Elle retourne le **numéro d'ordre du calque** dans la timeline (1 pour le calque le plus haut, 2 pour le suivant, etc.).

Sa puissance vient du fait que chaque calque possède un `index` unique. Il peut permettre par exemple d'appeler un calque par son index plutôt que son nom. On peu ainsi, facilement, appeler "le layer tout en haut de la comp" ou "le layer qui précède" ...

**Exemple : décaler le départ d'une animation sur chaque calque.**

Imaginons 5 calques dupliqués, tous avec la même expression sur leur Opacité :

```javascript
const delai = (index - 1) * 0.2;  // 0s, 0.2s, 0.4s, 0.6s, 0.8s
linear(time - delai, 0, 0.5, 0, 100)
// chaque calque apparaît 0.2 secondes après le précédent
```

> 💡 `index` est l'outil de base du **motion design procédural** : une seule expression appliquée à X calques dupliqués produit X comportements différents automatiquement.

<span id="keys"></span>
## numKeys, key(n), nearestKey()

Ces trois propriétés permettent d'**interroger les keyframes** d'une propriété directement depuis une expression : combien y en a-t-il, où se trouvent-elles, laquelle est la plus proche ?

### numKeys

Retourne le **nombre total de keyframes** sur la propriété courante.

```javascript
numKeys   // → 3 si la propriété a 3 keyframes
```

### key(n)

Retourne un **objet keyframe** à l'index `n` (commence à 1). Cet objet expose plusieurs propriétés :

```javascript
key(1).time     // le temps de la 1ère keyframe, en secondes
key(1).value    // la valeur de la 1ère keyframe
key(numKeys).time   // le temps de la dernière keyframe
```

**Exemple : déclencher un effet uniquement après la dernière keyframe.**

```javascript
const derniere = key(numKeys).time;
linear(time, derniere, derniere + 1, 0, 100)
// opacité passe de 0 à 100 dans la seconde qui suit la dernière keyframe
```

### nearestKey()

Retourne l'objet keyframe **le plus proche** du temps passé en argument.

```javascript
nearestKey(time).index   // index de la keyframe la plus proche de la tête de lecture
nearestKey(time).time    // temps de cette keyframe
```

> 💡 Ces outils sont utiles pour créer des expressions qui s'adaptent dynamiquement au contenu animé, sans avoir à coder en dur des valeurs de temps.

<span id="lier"></span>
## Lier des propriétés entre calques

Jusqu'ici, nos expressions lisaient des valeurs de la propriété elle-même (`value`, `time`). On peut aussi **aller chercher une valeur sur n'importe quel autre calque** de la composition.

> Attention : il n'est jamais question de **modifier** une propriété d'un autre calque depuis une expression. On ne peut qu'aller **lire** une valeur pour l'utiliser dans notre propre calcul.

### thisComp.layer()

La syntaxe pour accéder à un autre calque :

```javascript
thisComp.layer("Nom du calque")   // par nom, plus lisible,
                                  // attention aux coquilles : une lettre différente et l'expression est cassée
thisComp.layer(1)                 // par index, le calque le plus haut dans la timeline
```

**Exemple :** lier la rotation du calque courant à la position X d'un autre calque.

```javascript
const source = thisComp.layer("CONTROLS").transform.position[0];
// source = valeur X de la position du calque "CONTROLS"

linear(source, 0, 1920, -180, 180)
// traduit une position X (de 0 à 1920px) en rotation (de -180° à 180°)
```

**Exemple avec `valueAtTime()` :** un calque suit un autre avec du retard.

```javascript
thisComp.layer("Leader").transform.position.valueAtTime(time - 0.3)
// ce calque est toujours à la position qu'occupait "Leader" 0.3 secondes auparavant
```

> 💡 Combiné à `index`, cette technique permet de créer des chaînes de calques en cascade, où chacun suit le précédent avec un retard progressif, avec une seule expression commune à tous.
