## loopOut() / loopIn() <span id="array"></span>

`loopOut()` et `loopIn()` font une seule chose : **prolonger une animation existante au-delà de ses keyframes**, sans en ajouter de nouveaux. C'est AE qui répète la séquence à ta place.
La règle d'or : **il faut au minimum 2 keyframes** sur la propriété pour que la boucle ait quelque chose à répéter.

**Syntaxe complète :**

```
loopOut(type, numKeyframes)
// Les deux paramètres sont optionnels
// Par défaut : loopOut() est identique à loopOut("cycle", 0)
```

### Les 4 types 

```
loopOut("cycle")     // Répète la séquence du premier au dernier keyframe
loopOut("pingpong")  // Fait l'aller puis le retour, en miroir
loopOut("offset")    // Répète mais en décalant cumulativement la valeur
loopOut("continue")  // Ne répète pas — continue à la même vitesse indéfiniment
```

**cycle** — le plus courant. La séquence repart du début à chaque fois. Attention : si la valeur du dernier keyframe est différente de celle du premier, il y aura un saut visuel. Pour une boucle fluide, le dernier keyframe doit avoir la même valeur que le premier.

**pingpong** — la séquence joue en avant, puis en arrière, alternativement. Idéal pour un mouvement de balancier ou un effet de respiration.

**offset** — à chaque répétition, la valeur repart du point d'arrivée du cycle précédent. C'est le type le moins intuitif mais le plus puissant : parfait pour une roue qui tourne indéfiniment, ou un objet qui monte sans fin.

**continue** — ne répète rien. AE calcule la vélocité au dernier keyframe et continue dans cette direction. Utile pour simuler un objet qui sort du cadre naturellement.

### Le paramètre numKeyframes

C'est la source de confusion la plus fréquente avec `loopOut()`. **`numKeyframes` ne désigne pas un nombre de keyframes mais un nombre de segments.**

```
loopOut("cycle", 0)  // boucle sur TOUS les keyframes (défaut)
loopOut("cycle", 1)  // boucle sur le dernier segment → 2 keyframes inclus
loopOut("cycle", 2)  // boucle sur les 2 derniers segments → 3 keyframes inclus
```

> 💡 Règle simple : `numKeyframes = N` → `N+1` keyframes sont inclus dans la boucle.

### loopOutDuration()

Variante de `loopOut()` où la boucle est définie par une **durée en secondes** plutôt que par un nombre de segments.

```
loopOutDuration("cycle", 2)  // boucle sur les 2 dernières secondes d'animation
```

## wiggle() <span id="wiggle"></span>

`wiggle()` ajoute un **mouvement organique et aléatoire** à une propriété, sans aucun keyframe. À chaque frame, AE génère une valeur légèrement différente autour de la valeur courante de la propriété.

> 💡 Analogie : imagine que tu confies la propriété à quelqu'un qui a trop bu de café. Il maintient globalement la valeur mais ne reste jamais parfaitement immobile.

**Syntaxe complète :**

```
wiggle(freq, amp, octaves, amp_mult, t)
```

### Les paramètres dans l'ordre

```
wiggle(freq, amp)
// freq     → obligatoire. Nombre d'oscillations par seconde.
// amp      → obligatoire. Amplitude maximale du déplacement.
//            S'applique autour de la valeur courante, pas à partir de 0.
// octaves  → optionnel, défaut = 1. Couches de bruit superposées.
//            Plus la valeur est haute, plus le mouvement est détaillé et complexe.
// amp_mult → optionnel, défaut = 0.5. Vitesse à laquelle l'amplitude diminue
//            à chaque octave supplémentaire.
// t        → optionnel, défaut = time. Instant de référence pour le calcul.
```

**Exemples concrets :**

```
wiggle(5, 30)         // 5 oscillations/sec, amplitude de 30px — mouvement nerveux
wiggle(1, 50)         // 1 oscillation/sec, amplitude de 50px — mouvement lent et ample
wiggle(3, 20, 3)      // ajout de 3 octaves — mouvement plus organique et irrégulier
```

### Wiggle sur un seul axe

Par défaut, `wiggle()` appliqué à la Position bouge sur X **et** Y simultanément. Pour restreindre le mouvement à un seul axe, il faut comprendre deux concepts : les **Arrays** et les **variables**.

**Rappel sur les Arrays :** la propriété Position contient 2 valeurs — `[X, Y]`. Chaque valeur est accessible par son index, qui commence à 0 :

```
value[0]  // la valeur X de la propriété
value[1]  // la valeur Y de la propriété
```

**Rappel sur les variables :** une variable est une boîte qui stocke un résultat pour le réutiliser ailleurs dans l'expression. On la déclare avec `var`.

```
var w = wiggle(3, 40);
// w contient maintenant le résultat de wiggle — un Array [X_wiggly, Y_wiggly]

[value[0], w[1]]
// → garde la valeur X d'origine (pas de wiggle sur X)
// → remplace la valeur Y par la valeur Y du wiggle
```

### seedRandom() — un wiggle qui ne change pas au rendu

Par défaut, `wiggle()` génère un résultat **différent à chaque rendu**. C'est souvent ce qu'on veut, mais parfois on a besoin d'un wiggle déterministe — le même résultat à chaque fois.

```
seedRandom(42, true);
wiggle(3, 40)
// Le chiffre 42 est une "graine" arbitraire — la valeur n'a pas d'importance,
// seule sa constance compte. true = résultat stable entre les rendus.
```

## time et valueAtTime() <span id="valueattime"></span>

### time

`time` est une variable **disponible automatiquement** dans toute expression — on l'a déjà croisée dans le Module 1. Elle représente la position de la tête de lecture de la composition, exprimée en **secondes**.

```
time        // à la frame 0 → 0
            // à la frame 25 (comp 25fps) → 1
            // à la frame 50 → 2
```

Combiné à une opération mathématique, `time` devient un moteur d'animation sans keyframe :

```
// Sur la propriété Rotation :
time * 90   // tourne de 90° par seconde — soit un tour complet toutes les 4 secondes
```

### valueAtTime()

`valueAtTime(t)` lit la valeur d'une propriété **à un instant précis**, quel que soit la position actuelle de la tête de lecture. C'est une fenêtre dans le passé ou le futur de l'animation.

**Syntaxe :**

```
thisLayer.transform.position.valueAtTime(t)
// t → le temps en secondes auquel on veut lire la valeur
```

**Cas d'usage — le décalage temporel :** Imaginons un calque A dont la position est animée avec des keyframes. On veut qu'un calque B suive exactement le même chemin, mais avec 0,5 secondes de retard. Sur la propriété Position du calque B :

```
thisComp.layer("Calque A").transform.position.valueAtTime(time - 0.5)
// "quelle était la position de Calque A il y a 0,5 secondes ?"
```

> 💡 Cette technique permet de créer des effets de traîne ou écho, ou des éléments qui suivent un leader avec du retard — sans dupliquer une seule keyframe.

## linear() / ease() / easeIn() / easeOut() <span id="linear"></span>

Ces quatre fonctions font **une seule et unique chose** : prendre une valeur d'entrée dans une plage connue, et la **convertir proportionnellement** vers une plage de sortie. Ce qui les distingue, c'est la **courbe** de cette conversion.

> 💡 Analogie : c'est exactement ce que fait une règle de trois, mais avec la possibilité d'adoucir la transition.

### La syntaxe, argument par argument

```
linear(t, tMin, tMax, value1, value2)
//  t      → la valeur d'entrée (souvent time, ou value, ou n'importe quel nombre)
//  tMin   → la valeur minimale de t (début de la plage d'entrée)
//  tMax   → la valeur maximale de t (fin de la plage d'entrée)
//  value1 → la valeur de sortie quand t = tMin
//  value2 → la valeur de sortie quand t = tMax
```

**Exemple concret :** piloter l'opacité d'un calque entre les secondes 1 et 3 de la composition.

```
linear(time, 1, 3, 0, 100)
// quand time = 1  → opacité = 0
// quand time = 2  → opacité = 50 (milieu, progression linéaire)
// quand time = 3  → opacité = 100
// avant 1 et après 3 → AE clamp automatiquement (0 avant, 100 après)
```

### Les 4 variantes

```
linear(t, tMin, tMax, v1, v2)   // progression mécanique, vitesse constante
ease(t, tMin, tMax, v1, v2)     // ralentit au début ET à la fin (easeInOut)
easeIn(t, tMin, tMax, v1, v2)   // ralentit uniquement au début
easeOut(t, tMin, tMax, v1, v2)  // ralentit uniquement à la fin
```

**Raccourci :** si `t` est le seul argument qui varie et qu'on veut rester dans la plage `[0, 1]`, la syntaxe courte existe :

```
linear(t, v1, v2)   // équivalent à linear(t, 0, 1, v1, v2)
```

### t n'est pas forcément time

C'est la subtilité la plus importante. `t` peut être **n'importe quelle valeur numérique** — pas seulement le temps.

```
// Piloter la rotation depuis la valeur d'un Slider Control :
var slider = effect("Mon Slider")("Curseur");
linear(slider, 0, 100, -45, 45)
// quand le slider vaut 0   → rotation = -45°
// quand le slider vaut 100 → rotation = +45°
```

## Lier des propriétés entre calques <span id="lier"></span>

Jusqu'ici, nos expressions lisaient des valeurs de la propriété elle-même (`value`, `time`). On peut aussi **aller chercher une valeur sur n'importe quel autre calque** de la composition. ATTENTION, il n'est toujours pas question de MODIFIER une propriété autre que celle pour laquelle nous écrivons l'expression, UNIQUEMENT aller chercher une valeur pour l'utiliser dans notre expression.

### thisComp.layer()

La syntaxe pour accéder à un autre calque — déjà vue dans le Module 1, on la met maintenant en pratique réelle.

```
thisComp.layer("Nom du calque")   // par nom — plus lisible, attention aux coquilles
thisComp.layer(1)                 // par index — le calque le plus haut dans la timeline
```

**Exemple :** lier la rotation du calque courant à la position X d'un autre calque.

```
var source = thisComp.layer("CONTROLS").transform.position[0];
// source = valeur X de la position du calque "CONTROLS"

linear(source, 0, 1920, -180, 180)
// traduit une position X (de 0 à 1920px) en rotation (de -180° à 180°)
```

### effect() — le calque CONTROLS

En pratique, lier les propriétés directement entre calques est fragile : si tu renommes un calque ou modifies sa position, l'expression peut casser. La méthode professionnelle consiste à créer un **calque de contrôle dédié**, souvent appelé `CONTROLS`, sur lequel on place des **Expression Controllers** — des effets d'AE conçus spécifiquement pour exposer des valeurs pilotables par expression.

Les Expression Controllers disponibles dans AE (Effects → Expression Controls) :

* **Slider Control** → expose un nombre
* **Angle Control** → expose un angle
* **Checkbox Control** → expose un booléen (0 ou 1)
* **Color Control** → expose une couleur
* **Point Control** → expose une position 2D

**Syntaxe pour accéder à un Effect :**

```
thisComp.layer("CONTROLS").effect("Nom de l'effet")("Curseur")
//                                  ↑ nom de l'effet       ↑ nom du paramètre
```

**Exemple complet :** un seul Slider Control pilote la rotation de 3 calques différents. Sur la propriété Rotation de chacun des 3 calques, on écrit la même expression :

```
var ctrl = thisComp.layer("CONTROLS").effect("Vitesse de rotation")("Curseur");
ctrl * time
// le slider multiplie la vitesse — à 0, pas de rotation ; à 1, vitesse normale ; à 2, double vitesse
```

> 💡 C'est l'une des approches les plus puissantes des expressions : un seul point de contrôle pour piloter l'ensemble d'une composition. Modifier le slider suffit — pas besoin de toucher aux 3 calques individuellement.
