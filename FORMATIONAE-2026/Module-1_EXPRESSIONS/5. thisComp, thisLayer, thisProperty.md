## Les trois objets et leur hiérarchie 

Dans After Effects, toute expression s'exécute dans un contexte précis. Ce contexte est représenté par trois objets toujours disponibles, sans avoir à les déclarer :

```
thisComp          → la composition qui contient tout
  └── thisLayer   → le calque qui porte l'expression
        └── thisProperty  → la propriété sur laquelle l'expression est attachée
```

Ces trois objets sont le point de départ de la grande majorité des expressions. Comprendre à quoi chacun correspond évite de nombreuses erreurs et permet d'écrire des expressions qui **continuent de fonctionner même si on réorganise le projet**.

---
<span id="thiscomp"></span>
## thisComp 

`thisComp` représente **la composition dans laquelle l'expression s'exécute**. Il donne accès aux propriétés globales de la comp et à tous ses calques.

**Propriétés utiles :**

```javascript
thisComp.width        // largeur de la comp en pixels
thisComp.height       // hauteur de la comp en pixels
thisComp.duration     // durée totale en secondes
thisComp.frameDuration // durée d'une frame en secondes (1/framerate)
thisComp.numLayers    // nombre de calques dans la comp
thisComp.name         // nom de la composition (String)
```

**Accéder à un calque depuis thisComp :**

```javascript
thisComp.layer("MON CALQUE")      // par nom (sensible à la casse)
thisComp.layer(2)                 // par index (commence à 1 dans AE)
thisComp.layer(thisLayer.index - 1)  // le calque juste au-dessus
```

**Exemple concret : centrer dynamiquement sur la comp :**

```javascript
// Sur la propriété Position d'un calque
[thisComp.width / 2, thisComp.height / 2]
```

Ce calque sera toujours au centre, quelle que soit la taille de la comp, même si on change le format en cours de projet.

> 💡 `thisComp` est aussi le point d'entrée pour accéder à une autre composition depuis une expression.

---
<span id="thislayer"></span>
## thisLayer 

`thisLayer` représente **le calque qui porte l'expression**. C'est l'objet le plus utile pour écrire des expressions "portables", il ne dépend d'aucun nom, il désigne toujours "ce calque-ci". 

**Propriétés utiles :**

```javascript
thisLayer.name          // nom du calque (String)
thisLayer.index         // position du calque dans la timeline (commence à 1)
thisLayer.width         // largeur du calque source en pixels
thisLayer.height        // hauteur du calque source en pixels
thisLayer.inPoint       // temps d'entrée du calque en secondes
thisLayer.outPoint      // temps de sortie du calque en secondes
thisLayer.startTime     // décalage temporel du calque
```

**Accéder aux transformations du calque :**

```javascript
thisLayer.transform.position    // position courante [x, y]
thisLayer.transform.rotation    // rotation courante
thisLayer.transform.scale       // échelle courante [x, y]
thisLayer.transform.opacity     // opacité courante
```

**Exemple concret : décaler chaque calque selon son index :**

```javascript
// Sur la propriété Position
const decalage = (thisLayer.index - 1) * 80;
[value[0] + decalage, value[1]]
```

Si on duplique ce calque, chaque copie se placera automatiquement 80px plus à droite que la précédente, sans toucher à aucun keyframe.

**Exemple concret : faire disparaître un calque à sa sortie :**

```javascript
// Sur la propriété Opacité
const duree = thisLayer.outPoint - thisLayer.inPoint;
const tempsRestant = thisLayer.outPoint - time;
linear(tempsRestant, 0, 0.5, 0, 100)
```

> 💡 `thisLayer` sans point d'accès supplémentaire désigne le calque entier. Dans la plupart des expressions, on l'utilise en combinaison : `thisLayer.index`, `thisLayer.transform.position`, etc.

---
<span id="thisproperty"></span>
## thisProperty 
`thisProperty` représente **la propriété sur laquelle l'expression est attachée**. C'est l'objet le plus rarement utilisé des trois, mais il permet des choses impossibles autrement, notamment inspecter les keyframes de la propriété elle-même depuis l'expression.

**Propriétés et méthodes utiles :**

```javascript
thisProperty.numKeys              // nombre de keyframes sur cette propriété
thisProperty.key(1)               // premier keyframe
thisProperty.key(n).time          // temps du n-ième keyframe en secondes
thisProperty.key(n).value         // valeur du n-ième keyframe
thisProperty.valueAtTime(t)       // valeur de la propriété au temps t
```

**Exemple concret : récupérer le temps du dernier keyframe :**

```javascript
const dernierKey = thisProperty.key(thisProperty.numKeys);
dernierKey.time    // temps en secondes du dernier keyframe
```

**Exemple concret : interpoler depuis la valeur du keyframe précédent :**

```javascript
// Trouver le keyframe actif juste avant la tête de lecture
const k = 1;
while (k < thisProperty.numKeys && thisProperty.key(k).time < time) {
    k++;
}
const keyPrecedent = thisProperty.key(Math.max(1, k - 1));
linear(time, keyPrecedent.time, keyPrecedent.time + 0.5, 0, 100)
```

> 💡 `thisProperty.valueAtTime(t)` est particulièrement utile pour créer des effets d'écho ou de décalage temporel : on lit la valeur que *cette même propriété* avait à un instant passé, et on l'utilise comme valeur courante.

<span id="references"></span>
## Pourquoi éviter les références par nom 

Référencer un calque par son nom est la méthode la plus naturelle, mais aussi la plus fragile :

```javascript
thisComp.layer("TITRE")    // casse si le calque est renommé "Titre" ou "titre"
```

**Les alternatives plus robustes :**

**Par index**, utile quand la position dans la timeline est stable :
```javascript
thisComp.layer(3)    // toujours le 3ème calque depuis le haut
```

**Par rapport à thisLayer**, la plus portable de toutes :
```javascript
thisComp.layer(thisLayer.index - 1)    // le calque juste au-dessus
thisComp.layer(thisLayer.index + 1)    // le calque juste en-dessous
```

**Via un effet de contrôle**, la méthode recommandée pour les rigs complexes :
```javascript
// Un Slider Control sur ce calque pilote une valeur
effect("Vitesse")("Curseur")
```

Dans ce cas l'expression ne référence aucun autre calque, elle lit un effet sur elle-même. C'est la méthode la plus robuste : le calque peut être renommé, déplacé, dupliqué, l'expression continue de fonctionner.

> 💡 Règle pratique : si une expression doit survivre à une réorganisation du projet, préfère `thisLayer` et les effets de contrôle aux références par nom. Garde les références par nom pour les expressions rapides et temporaires.

<span id="pratiques"></span>
## Cas pratiques 

**Centrer un élément sur la comp, quelles que soient ses dimensions :**

```javascript
// Sur la propriété Position
[thisComp.width / 2, thisComp.height / 2]
```

**Répartir des calques en ligne automatiquement selon leur index :**

```javascript
// Sur la propriété Position, même expression sur tous les calques
const espacement = 120;
const total = thisComp.numLayers;
const startX = (thisComp.width - (total - 1) * espacement) / 2;

[startX + (thisLayer.index - 1) * espacement, thisComp.height / 2]
```

**Créer un décalage temporel (delay) basé sur l'index du calque :**

```javascript
// Sur n'importe quelle propriété animée avec loopOut par exemple
const delai = (thisLayer.index - 1) * 0.1;    // 100ms entre chaque calque
thisComp.layer(1).transform.position.valueAtTime(time - delai)
```

**Lire la durée du calque pour piloter une animation interne :**

```javascript
// Sur la propriété Opacité, fondu sortant sur la dernière seconde
const debut = thisLayer.outPoint - 1;
linear(time, debut, thisLayer.outPoint, 100, 0)
```
