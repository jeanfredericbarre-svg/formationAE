## t comme repère temporel

Dans la documentation Adobe et dans la grande majorité des expressions que tu rencontreras, tu verras souvent la lettre `t` apparaître comme argument. Ce n'est pas un mot-clé réservé d'AE — c'est une **convention de nommage universelle** pour désigner un argument de type `Number` représentant un temps en secondes.

> 💡 Tu pourrais très bien l'appeler `temps`, `x`, ou `banane`. La convention `t` s'est imposée parce qu'elle est courte et parlante. Respecte-la : tes expressions seront lisibles par n'importe qui.

Dans son rôle le plus courant, `t` représente la **position de la tête de lecture** de la composition. AE expose cette valeur automatiquement via `time`.

La majorité des méthodes AE qui acceptent `t` ont `time` comme valeur par défaut, ce qui signifie qu'on peut les appeler **sans argument** :

```
wiggle(3, 40)           // identique à wiggle(3, 40, 1, 0.5, time)
smooth()                // identique à smooth(0.2, 5, time)
loopOut()               // identique à loopOut("cycle", 0)
```

Les méthodes pour lesquelles `t` est **obligatoire** :

| Méthode | Signature | t obligatoire ? |
| --- | --- | --- |
| `valueAtTime()` | `valueAtTime(t)` | ✅ oui |
| `velocityAtTime()` | `velocityAtTime(t)` | ✅ oui |
| `speedAtTime()` | `speedAtTime(t)` | ✅ oui |
| `nearestKey()` | `nearestKey(t)` | ✅ oui |
| `wiggle()` | `wiggle(freq, amp, octaves, amp_mult, t)` | non — défaut `time` |
| `temporalWiggle()` | `temporalWiggle(freq, amp, octaves, amp_mult, t)` | non — défaut `time` |
| `smooth()` | `smooth(width, samples, t)` | non — défaut `time` |

## t comme driver d'interpolation

C'est là que `t` devient vraiment puissant. Pour les fonctions `linear()`, `ease()`, `easeIn()` et `easeOut()`, `t` n'est **pas forcément du temps** — c'est simplement **la valeur d'entrée** du remapping.

```
linear(t, tMin, tMax, value1, value2)
```

`t` peut être :

* `time` → interpolation temporelle classique
* `value` → remapping d'une plage de valeurs vers une autre
* le résultat de n'importe quelle expression numérique

```
// t = time → fondu d'entrée entre 0 et 2 secondes
linear(time, 0, 2, 0, 100)

// t = value → convertir un slider (0–100) en rotation (-180° à 180°)
var s = effect("Mon Slider")("Curseur");
linear(s, 0, 100, -180, 180)

// t = une position X → piloter l'opacité par la position horizontale
var px = thisLayer.transform.position[0];
linear(px, 0, thisComp.width, 0, 100)
```

> 💡 Comprendre que `t` est juste "une valeur d'entrée quelconque" déverrouille toute la puissance de `linear()` et `ease()`. Ces fonctions ne servent pas qu'à animer dans le temps — elles servent à **traduire n'importe quelle valeur vers n'importe quelle autre**.

### Les 4 variantes

```
linear(t, tMin, tMax, v1, v2)   // progression mécanique, vitesse constante
ease(t, tMin, tMax, v1, v2)     // ralentit au début ET à la fin (easeInOut)
easeIn(t, tMin, tMax, v1, v2)   // ralentit uniquement au début
easeOut(t, tMin, tMax, v1, v2)  // ralentit uniquement à la fin
```

**Raccourci :** si `t` est compris entre 0 et 1, la syntaxe courte existe :

```
linear(t, v1, v2)   // équivalent à linear(t, 0, 1, v1, v2)
```

## Normaliser time — la progression du calque

Un pattern extrêmement courant dans les expressions professionnelles : ramener `time` (valeur absolue en secondes, imprévisible) à une valeur **entre 0 et 1** représentant la progression du calque dans la timeline.

```
var t = (time - inPoint) / (outPoint - inPoint);
// t = 0 au début du calque, t = 1 à la fin du calque
ease(t, 0, 1, valeurDébut, valeurFin)
```

Ce pattern est puissant parce qu'il rend l'expression **indépendante de la position du calque** dans la timeline. Tu peux déplacer le calque librement — l'animation reste toujours calée sur sa durée propre.

### Exemple complet — fondu lié à la durée du calque

```
// Sur la propriété Opacity d'un calque :
var t = (time - inPoint) / (outPoint - inPoint);
ease(t, 0, 1, 0, 100)
// Fondu de 0% à 100% sur toute la durée du calque, peu importe où il est placé
```

### Exemple — fondu entrant sur les 20% du calque seulement

```
var t = (time - inPoint) / (outPoint - inPoint);
ease(t, 0, 0.2, 0, 100)
// Fondu de 0% à 100% sur le premier cinquième du calque, puis reste à 100%
```

> 💡 Combiner la normalisation de `time` avec `ease()` est l'un des patterns les plus utilisés dans les expressions professionnelles. C'est la base des systèmes d'animation procédurale dans AE.
