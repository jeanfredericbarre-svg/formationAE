<span id="pourquoi"></span>
## Pourquoi ces effets existent

Quand on écrit une expression, la tentation naturelle est de coder les valeurs directement dans le code :

```javascript
wiggle(3, 25)
```

Ça fonctionne, mais si on veut ajuster la fréquence ou l'amplitude, il faut rouvrir l'éditeur d'expressions, modifier le code, fermer. Sur un projet avec des dizaines d'expressions, c'est vite pénible.

Les **Expression Controls** résolvent ce problème. Ce sont des effets sans rendu visuel, dont l'unique rôle est d'**exposer un paramètre ajustable dans le panneau Effets**, lisible par une expression. Le code ne change plus, seul le curseur bouge.

```javascript
// Avant : valeurs figées dans le code
wiggle(3, 25)

// Après : valeurs exposées comme paramètres
const freq = effect("Fréquence")("Slider");
const amp  = effect("Amplitude")("Slider");
wiggle(freq, amp)
```

Dans cet exemple, l'utilisateur final règle `Fréquence` et `Amplitude` dans le panneau Effets comme n'importe quel effet, sans jamais toucher au code.

> 💡 Les Expression Controls se trouvent dans **Effets > Expression Controls**. Ils ne produisent aucun rendu visuel sur le calque, ils n'existent que pour être lus par des expressions.

<span id="types"></span>
## Les six types disponibles

Chaque type correspond à une nature de valeur différente.

**Slider Control**

Expose un nombre. C'est le type le plus utilisé pour des vitesses, amplitudes, durées, décalages, …

```javascript
effect("Vitesse")("Slider")    // retourne un Number, ex : 3.5
```

**Angle Control**

Expose un angle en degrés. Identique à un Slider en pratique, mais l'interface affiche un cadran rotatif, plus intuitif pour une valeur angulaire.

```javascript
effect("Direction")("Angle")    // retourne un Number en degrés, ex : 45
```

**Checkbox Control**

Expose un booléen : 0 (décoché) ou 1 (coché). Idéal pour activer ou désactiver un comportement.

```javascript
effect("Mode actif")("Checkbox")    // retourne 0 ou 1
```

**Color Control**

Expose une couleur. Retourne un tableau `[r, g, b, a]` (a c'est pour l'alpha) avec des valeurs entre 0 et 1.

```javascript
effect("Couleur principale")("Color")    // retourne [r, g, b, a]
```

**Point Control**

Expose une position dans la composition. Retourne un tableau `[x, y]` en pixels.

```javascript
effect("Point d'origine")("Point")    // retourne [x, y]
```

**Layer Control**

Expose une référence à un calque de la composition. Retourne un objet calque, utilisable comme `thisComp.layer(...)`.

```javascript
effect("Calque cible")("Layer")    // retourne un objet Layer
effect("Calque cible")("Layer").transform.position    // position de ce calque
```

> 💡 Le **Layer Control** est le seul Expression Control qui ne retourne pas une valeur brute mais un objet. Il permet de désigner dynamiquement un calque source sans coder son nom ou son index dans l'expression.

<span id="appliquer"></span>
## Appliquer un Expression Control

Un Expression Control s'applique comme n'importe quel effet : sélectionner le calque, puis **Effets > Expression Controls > Slider Control** (ou le type voulu).

**Nommer l'effet immédiatement**

Le nom affiché dans le panneau Effets est celui qu'on utilisera dans `effect("Nom")`. Renommer l'effet dès l'application évite les confusions :

```
Double-clic sur le nom de l'effet dans le panneau Effets → renommer
```

**Sur quel calque l'appliquer ?**

Deux approches courantes :

- **Sur le calque porteur de l'expression** : simple, tout est au même endroit. Convient pour les paramètres qui ne concernent qu'un seul calque.
- **Sur un Null dédié** : recommandé dès que plusieurs calques partagent les mêmes paramètres. Voir plus loin : [Null contrôleur](#null-controleur).

> 💡 On peut empiler autant d'Expression Controls que nécessaire sur un même calque. Il est courant de voir un Null "CONTROLEUR" avec une dizaine de Sliders, un Color Control et une Checkbox, tous nommés explicitement.

<span id="lire"></span>
## Lire un Expression Control depuis une expression

La syntaxe est celle de `effect()` : nom de l'effet, puis nom de la propriété du contrôle.

```javascript
effect("Nom du contrôle")("Slider")      // Slider Control
effect("Nom du contrôle")("Angle")       // Angle Control
effect("Nom du contrôle")("Checkbox")    // Checkbox Control
effect("Nom du contrôle")("Color")       // Color Control
effect("Nom du contrôle")("Point")       // Point Control
effect("Nom du contrôle")("Layer")       // Layer Control
```

Sans préciser de calque, `effect()` cherche sur `thisLayer` le calque qui porte l'expression :

```javascript
// Expression sur le calque qui porte aussi le Slider
effect("Amplitude")("Slider")
```

Pour lire un Expression Control sur un autre calque :

```javascript
// Expression sur un calque B, qui lit un Slider sur le calque A
thisComp.layer("CONTROLEUR").effect("Amplitude")("Slider")
```

> 💡 Les noms des propriétés (`"Slider"`, `"Color"`, `"Point"`…) correspondent aux noms affichés dans le panneau Effets en version **anglaise** d'AE. En version française, ces noms peuvent différer : `"Curseur"`, `"Couleur"`, `"Point"`. Pour éviter les problèmes de compatibilité multilingue, préférer l'accès par index : `effect("Amplitude")(1)`.

<span id="null-controleur"></span>
## Le Null contrôleur

L'idée : un **calque Null dédié** regroupe tous les Expression Controls du projet. Tous les controles sont regroupés au même endroit. C'est aussi un gros avantage si vous devez créer un MOGRT, tous les contrôles sont là.

**Mise en place :**

1. Créer un calque Null : **Calque > Nouveau > Objet nul**
2. Le renommer `CONTROLEUR` (ou `_CTRL`, `RIG`, selon la convention du projet)
3. Appliquer tous les Expression Controls nécessaires sur ce Null
4. Les nommer explicitement
5. Dans les expressions des autres calques, référencer ce Null

```javascript
// Sur n'importe quel calque de la comp
const ctrl = thisComp.layer("CONTROLEUR");

const freq    = ctrl.effect("Fréquence")("Slider");
const amp     = ctrl.effect("Amplitude")("Slider");
const couleur = ctrl.effect("Couleur scène")("Color");
const actif   = ctrl.effect("Animation active")("Checkbox");
```

**Avantages :**

- Tous les paramètres du projet sont visibles au même endroit
- Modifier une valeur affecte tous les calques qui la lisent simultanément
- Le Null peut être verrouillé, mis en solo, masqué sans affecter le rendu

> 💡 Nommer le calque avec un préfixe spécial (`_` ou des majuscules) le fait remonter en haut de la timeline et le distingue visuellement des calques de contenu. C'est une convention largement répandue.

**Wiggle entièrement paramétrable :**

```javascript
// Sur la propriété Position
// Deux Sliders sur ce calque : "Fréquence" et "Amplitude"
const freq = effect("Fréquence")("Slider");
const amp  = effect("Amplitude")("Slider");
wiggle(freq, amp)
```

**Activation d'un comportement via une Checkbox :**

```javascript
// Sur la propriété Position
const actif = effect("Wiggle actif")("Checkbox");
actif ? wiggle(4, 20) : value
```

**Fondu entrant dont la durée est réglable :**

```javascript
// Sur la propriété Opacité
// Un Slider "Durée fondu" sur ce calque, valeur en secondes
const duree = effect("Durée fondu")("Slider");
linear(time, thisLayer.inPoint, thisLayer.inPoint + duree, 0, 100)
```

**Couleur centralisée partagée entre plusieurs calques :**

```javascript
// Sur la propriété Couleur d'un solid
// Un Color Control "Couleur principale" sur le calque CONTROLEUR (index 1)
thisComp.layer(1).effect("Couleur principale")("Color")
```

**Mapper un Slider sur une plage personnalisée :**

```javascript
// Un Slider de 0 à 100 pilote une rotation de -180° à +180°
const v = effect("Rotation")("Slider");
linear(v, 0, 100, -180, 180)
```

**Suivre dynamiquement un calque désigné via un Layer Control :**

```javascript
// Sur la propriété Position
// Un Layer Control "Calque cible" sur ce calque
const cible = effect("Calque cible")("Layer");
cible.transform.position
```
