## C'est quoi Trim Paths ? 

Trim Paths est un opérateur de Shape Layer qui **révèle ou masque progressivement le contour d'une forme**, en définissant un point de début et un point de fin sur le tracé. Tout ce qui est entre ces deux points est visible ; le reste est invisible.

> 💡 Analogie : imagine que tu dessines à la main sur une feuille. Trim Paths simule exactement ça, le trait apparaît progressivement, comme si un stylo se déplaçait le long du contour.

Trim Paths ne fonctionne que sur le **contour** (Stroke) d'une forme, pas sur le fond. Assure-toi que ton groupe a bien un opérateur Contour actif.

**Pour l'ajouter :** dans la timeline du Shape Layer, sélectionne un groupe → bouton **Ajouter** → **Tracés de rognage**.

---
<span id="parametres"></span>
## Les paramètres 

```
Début     → point de départ du segment visible (0% = début du tracé)
Fin       → point d'arrivée du segment visible (100% = fin du tracé)
Décalage  → déplace les deux points simultanément le long du tracé
              sans changer la longueur du segment visible
```

**Configuration de départ typique :**

```
Début    = 0%
Fin      = 0%    ← rien n'est visible
Décalage = 0°
```

Pour révéler le tracé : animer **Fin** de 0% à 100%.

Pour effacer un tracé déjà visible : animer **Début** de 0% à 100% (pendant que Fin reste à 100%).

**Le Décalage** est particulièrement utile pour créer des effets de "point qui court" le long d'un tracé, on fixe un petit écart entre Début et Fin (par exemple 10%) et on anime le Décalage.

<span id="animer"></span>
## Animer un tracé qui se dessine 

**Cas 1 : dessin simple :**

```
Début   = 0%          (fixe)
Fin     = 0% → 100%   (animé sur la durée souhaitée)
```

Ajoute des keyframes avec `easeOut` sur la valeur de Fin pour un démarrage rapide qui ralentit à la fin, plus naturel qu'un mouvement linéaire.

**Cas 2 : écriture puis effacement :**

```
Frame 0  : Début = 0%,   Fin = 0%
Frame 60 : Début = 0%,   Fin = 100%   ← tracé entièrement dessiné
Frame 90 : Début = 100%, Fin = 100%   ← tracé entièrement effacé
```

**Cas 3 : segment qui se déplace (point courant) :**

```
Début    = 0%           (fixe)
Fin      = 10%          (fixe — seuls 10% du tracé sont visibles)
Décalage = 0° → 360°   (animé — fait tourner le segment sur le tracé)
```

> 💡 `Décalage` s'exprime en degrés (0° à 360° = un tour complet du tracé), pas en pourcentage. C'est un choix d'Adobe qui surprend au début.

<span id="expressions"></span>
## Trim Paths et expressions 

Tous les paramètres de Trim Paths acceptent des expressions, ce qui permet de les piloter dynamiquement.

**Exemple : synchroniser le dessin avec la durée du calque :**

```javascript
// Sur la propriété "Fin" du Trim Paths :
const t = (time - inPoint) / (outPoint - inPoint);
ease(t, 0, 1, 0, 100)
// Le tracé se dessine sur toute la durée du calque, avec lissage
// Déplace ou étire le calque : le tracé s'adapte automatiquement
```

**Exemple : Décalage piloté par un Slider Control :**

```javascript
// Sur la propriété "Décalage" :
thisComp.layer("CONTROLS").effect("Vitesse tracé")("Curseur").value * time
// Le slider contrôle la vitesse à laquelle le point court le long du tracé
```

**Trim Paths sur plusieurs formes dans le même groupe :**

Un seul opérateur Trim Paths s'applique à **toutes les formes du groupe**. Le mode **Simultanément** anime toutes les formes ensemble ; le mode **Individuellement** les anime en séquence, l'une après l'autre.

```
Trim Paths → Mode = Individuellement
→ chaque forme se dessine à la suite de la précédente
→ effet "dessin qui progresse forme par forme"
```
